# Что делать дальше (все сервисы подняты)

Все сервисы отвечают (kb, sales-agent, amocrm-api, n8n, site). Ниже — минимальный порядок действий.

---

## 1. Проверить n8n workflow и credential

1. Откройте **http://localhost:5678** → workflow **Sales Agent - KB Integration** (или импортируйте заново из `n8n/workflows/sales-agent-kb-integration-localhost.json`).
2. Убедитесь, что **Active** включён (зелёный).
3. В узле **LLM ProxyAPI** выберите credential **Header Auth** (ProxyAPI) с ключом для proxyapi.ru.
4. Сохраните (Ctrl+S).

---

## 2. Прогнать чат с сайта

1. Откройте **http://localhost:8000/site-integration-example.html**.
2. Нажмите на карточку (например «Дом 150 м²») → **Узнать условия** — откроется чат с контекстом карточки.
3. Напишите, например: «Меня интересует этот дом, хочу подробности».
4. Должен прийти ответ по базе знаний, с приглашением оставить контакт для персонального расчёта.
5. Напишите: «Меня зовут Иван, +79161234567» — ответ «Спасибо, мы свяжемся» и в AmoCRM должен создаться тестовый лид (если amocrm-api на 8010 доступен для n8n).

---

## 3. Проверить создание лида в AmoCRM

1. В n8n: **Executions** → последний запуск → убедитесь, что прошли узлы **Maybe Create Lead** → **Create Lead If Phone** → **AmoCRM Create Lead** (при сообщении с телефоном).
2. В AmoCRM: найдите сделку с префиксом **\[BRATS-TEST]** и тегом **brats_test** (или отфильтруйте по тегу). Должны быть контакт и примечание с текстом из чата.

Если лиды не попадают в нужную воронку — в **config.env** задайте **AMOCRM_TEST_PIPELINE_ID** (например 10140478) и перезапустите amocrm-api (или `./start_all_services.sh restart`).

---

## 4. Опционально: календарь показов и шаблоны

- В PostgreSQL применены миграции **07** и **08** (календарь `viewing_slots`, шаблоны `document_templates`, образцы продуктов). При необходимости выполните `db-init/init_all.sh` или только `07` и `08`.
- Дальше можно: в n8n при «запись на просмотр» читать свободные слоты из БД и создавать запись в `viewing_slots` + примечание к лиду в AmoCRM.

---

## 5. Дальнейшие шаги по продукту

- **Повторный вход (контакт известен):** передавать с сайта/бэкенда флаг `metadata.contact_known`, чтобы агент вёл до показа и дальше по воронке (см. AGENT_FUNNEL_STRATEGY.md).
- **Сохранять диалоги в PostgreSQL** (conversations/messages) — отдельный узел или сервис при желании иметь историю.
- **Синхронизация лидов:** доработать использование тестовой воронки и постепенно вводить те же стандарты в боевые воронки (см. STANDARDS_IMPLEMENTATION_PLAN.md).

Итого: убедиться, что workflow активен и credential выбран → прогнать чат с карточки и с передачей телефона → проверить появление тестового лида в AmoCRM → при необходимости настроить воронку и БД.
