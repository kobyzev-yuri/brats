# Отладка чата с сайта (step by step)

Цепочка: **сайт → прокси :8000 → n8n webhook → KB Search → Format → Sales Agent Call → ответ в чат.**

**n8n без Docker (рекомендуется):** запустите `./start_all_services.sh start` — n8n поднимется локально (npx), тогда в workflow используйте **localhost** для KB и агента. Импортируйте `workflows/sales-agent-kb-integration-localhost.json`.

**Где локальный n8n:** тот же URL — **http://localhost:5678**. Данные: `~/.n8n` (как у Docker — без повторной регистрации). Только n8n вручную: `cd n8n && ./start_n8n_local.sh`.

## Шаг 1: Workflow в n8n активен

1. Откройте n8n: http://localhost:5678
2. Откройте workflow **«Sales Agent - KB Integration»**
3. Убедитесь, что переключатель **Active** (вверху справа) **включён** (зелёный).  
   Если выключен — включите: иначе webhook не примет запрос.

## Шаг 2: Дёргаем чат с сайта

1. Откройте страницу: http://localhost:8000/site-integration-example.html
2. В блоке «Чат с нейропродажником» оставьте **«n8n flow (агент + CRM)»** и URL прокси:  
   `http://localhost:8000/api/n8n-proxy` (подставится сам, если открыли с порта 8000)
3. Нажмите кнопку чата (💬 внизу справа) или «Узнать условия» на любой карточке
4. В поле ввода напишите **«Привет»** и нажмите **Отправить**

Ожидание: в чате должен появиться ответ агента (или сообщение об ошибке, если что-то упало).

## Шаг 3: Смотрим выполнение в n8n

1. В n8n откройте **Executions** (левое меню)
2. Найдите **последнее** выполнение (самое верхнее)
3. Кликните по нему и посмотрите узлы:
   - **Webhook** — зелёный, во входе ваше `message: "Привет"`
   - **Normalize Input** — зелёный, на выходе `message`, `channel`, `target_audience`
   - **KB Search** — если **красный**, проверьте URL в узле: должен быть `http://localhost:8001/api/kb/search` (workflow **sales-agent-kb-integration-localhost.json**).
   - **Format KB Context** — зелёный, если KB Search прошёл
   - **Sales Agent Call** — зелёный, ответ агента
   - **Prepare Response** → **Respond to Webhook** — зелёные, ответ ушёл на сайт

Если какой-то узел **красный** — откройте его и прочитайте текст ошибки. Дальше:
- **KB Search** падает → см. **DEBUG_KB_SEARCH.md** (при локальном n8n в узле должен быть URL `http://localhost:8001/api/kb/search`, импорт: **sales-agent-kb-integration-localhost.json**).
- **Sales Agent Call** падает → проверьте, что sales-agent на 8003 отвечает: `curl -s http://localhost:8003/`

### Проверить, что запрос реально ушёл в KB и что вернулось

1. В **Executions** откройте выполнение и кликните узел **KB Search**.
2. Во вкладке **Input** (или **Request**) посмотрите тело запроса: поле **query** должно совпадать с текстом пользователя (например «Меня интересует дом 80м с черновой отделкой»). Так вы убедитесь, что запрос в KB идёт.
3. Во вкладке **Output** (или **Response**) посмотрите ответ: **results** — массив найденных кусков; если он пустой при вопросе про «дом 80 м»/«черновую отделку», в workflow понижен **min_similarity** до 0.35 и **limit** до 10 (файл **sales-agent-kb-integration-localhost.json**). Перезагрузите workflow и повторите тест.

## Шаг 4: Повторный тест (по желанию)

- В чате на сайте отправьте ещё одно сообщение (например «Какие есть варианты отделки?»).
- Снова откройте **Executions** и проверьте новое выполнение — все узлы должны быть зелёные, ответ в чате — осмысленный.

---

**Итого:** включить workflow → открыть сайт на :8000 → отправить «Привет» в чат → в n8n Executions проверить, что цепочка прошла до конца (все узлы зелёные).

## Контекст карточки и подсказки аналитики

Если пользователь открыл чат по кнопке на карточке объекта (например «Дом 150 м²», 9 500 000 ₽, участок 8 соток), сайт отправляет в webhook **metadata.real_estate_context** (название, цена, отделка, участок, ипотека, рассрочка, скидка). Workflow:

- **Normalize Input** формирует строку **additional_context** из этого объекта (и при необходимости из **metadata.analyst_hint** — подсказка аналитика на будущее).
- **KB Search** получает в запросе не только текст сообщения, но и этот контекст (поиск по базе учитывает объект и условия).
- **Format KB Context** подставляет в промпт LLM блок «Контекст от канала/аналитики», чтобы модель не переспрашивала уже известное и отвечала с учётом карточки.

В карточках на сайте в **data-product** можно передавать: name, price, finishing, land_area, mortgage, installment, discount_note, notes.
