# Отладка пустого ответа workflow

## Проблема: Workflow возвращает пустой ответ

Если workflow активирован, но возвращает пустой ответ, нужно проверить выполнение.

## Шаг 1: Проверьте Executions в n8n

1. Откройте n8n: http://localhost:5678
2. Перейдите в **Executions** (в левом меню)
3. Найдите последнее выполнение workflow **"amoCRM - Get Deal History (Simple)"**
4. Откройте его (кликните на выполнение)

## Шаг 2: Проверьте каждый узел

Проверьте статус каждого узла:

### Узел "Webhook" (зеленый ✅)
- **Input**: должен содержать `{"contact_id": 46692527}`
- **Output**: должен содержать те же данные

### Узел "Prepare Request" (зеленый ✅)
- **Input**: данные из Webhook
- **Output**: должен содержать:
  ```json
  {
    "contact_id": 46692527,
    "closed_statuses": [142, 143],
    "auth_header": "Bearer ...",
    "leads_url": "https://innovatoryclub.amocrm.ru/api/v4/leads?filter[contacts][id]=46692527"
  }
  ```

**Если этот узел красный ❌:**
- Проверьте сообщение об ошибке
- Возможно проблема с переменными окружения `AMOCRM_ACCESS_TOKEN` или `AMOCRM_SUBDOMAIN`

### Узел "Get Leads" (зеленый ✅)
- **Input**: данные из "Prepare Request"
- **Output**: должен содержать ответ от amoCRM API:
  ```json
  {
    "_embedded": {
      "leads": [...]
    }
  }
  ```

**Если этот узел красный ❌:**
- Проверьте сообщение об ошибке
- Возможно проблема с авторизацией (401)
- Возможно проблема с URL или фильтром

**Если этот узел возвращает пустой ответ:**
- Проверьте, что в ответе есть `_embedded.leads`
- Возможно API вернул статус 204 (No Content)

### Узел "Filter and Format" (зеленый ✅)
- **Input**: ответ от "Get Leads"
- **Output**: должен содержать отформатированную историю

**Если этот узел красный ❌:**
- Откройте узел и посмотрите на сообщение об ошибке
- Возможно проблема с доступом к данным через `$('prepare-request')`
- Возможно ошибка в JavaScript коде

**Если этот узел возвращает пустой `deal_history`:**
- Проверьте поле `debug` в ответе - там будет информация о том, почему сделки не найдены
- Возможно у контакта нет сделок со статусами 142, 143

### Узел "Respond" (зеленый ✅)
- **Input**: данные из "Filter and Format"
- **Output**: ответ отправлен клиенту

**Если этот узел красный ❌:**
- Проверьте сообщение об ошибке
- Возможно проблема с форматом ответа

## Шаг 3: Включите сохранение выполнений

Если выполнений нет в списке:

1. Откройте workflow **"amoCRM - Get Deal History (Simple)"**
2. Нажмите на три точки (⋮) в правом верхнем углу
3. Выберите **Settings**
4. Включите:
   - ✅ **Save Execution Data** - для сохранения всех выполнений
   - ✅ **Save Data on Error** - для сохранения ошибок
5. Сохраните изменения

## Шаг 4: Проверьте логи n8n

```bash
docker logs n8n-brats --tail 100 | grep -i -E "(error|deal-history|amocrm)"
```

## Шаг 5: Проверьте переменные окружения

В n8n Code node добавьте временный узел для проверки:

```javascript
return {
  json: {
    amocrm_subdomain: $env.AMOCRM_SUBDOMAIN,
    amocrm_token_set: !!$env.AMOCRM_ACCESS_TOKEN,
    token_length: $env.AMOCRM_ACCESS_TOKEN ? $env.AMOCRM_ACCESS_TOKEN.length : 0
  }
};
```

## Частые проблемы

### 1. Переменные окружения не доступны
**Симптом:** Узел "Prepare Request" падает с ошибкой "AMOCRM_ACCESS_TOKEN не найден"

**Решение:** 
- Проверьте, что переменные установлены в n8n: **Settings** → **Environment Variables**
- Или проверьте `config.env` и перезапустите n8n

### 2. Ошибка доступа к данным предыдущего узла
**Симптом:** Узел "Filter and Format" падает с ошибкой при вызове `$('prepare-request')`

**Решение:**
- Используйте данные напрямую из `$input` вместо `$('prepare-request')`
- Или передавайте `contact_id` через все узлы

### 3. API возвращает пустой ответ
**Симптом:** Узел "Get Leads" возвращает пустой объект или статус 204

**Решение:**
- Проверьте, что фильтр правильный: `filter[contacts][id]=46692527`
- Проверьте, что у контакта есть сделки

### 4. Workflow выполняется, но не возвращает ответ
**Симптом:** Все узлы зеленые, но curl получает пустой ответ

**Решение:**
- Проверьте узел "Respond" - возможно проблема с форматом ответа
- Убедитесь, что `responseBody` установлен как `={{ $json }}`

## Быстрый тест

Создайте простой тестовый workflow для проверки:

1. Импортируйте `amocrm-get-deal-history-test.json`
2. Активируйте его
3. Вызовите: `curl -X POST http://localhost:5678/webhook/amocrm-get-deal-history-test -H "Content-Type: application/json" -d '{"contact_id": 46692527}'`
4. Если этот workflow работает, значит проблема в основном workflow

## Что проверить в основном workflow

1. ✅ Workflow активирован
2. ✅ Все узлы выполняются без ошибок
3. ✅ Узел "Get Leads" возвращает данные от API
4. ✅ Узел "Filter and Format" правильно обрабатывает данные
5. ✅ Узел "Respond" получает данные и отправляет ответ
