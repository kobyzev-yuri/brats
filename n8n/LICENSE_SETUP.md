# Настройка лицензионного ключа n8n

## Сохранение license key

Ваш лицензионный ключ n8n: `bab52b80-51e4-4035-9a81-16c2b10637f0`

⚠️ **Важно**: Если вы уже активировали ключ через ссылку из почты, он уже привязан к вашему экземпляру n8n. Не нужно активировать его повторно - просто проверьте статус лицензии в настройках n8n.

### Шаг 1: Добавьте ключ в config.env (для справки)

Отредактируйте `/projects/brats/config.env`:

```bash
cd /projects/brats
nano config.env
```

Добавьте или обновите строку:
```bash
N8N_LICENSE_KEY=bab52b80-51e4-4035-9a81-16c2b10637f0
```

### Шаг 2: Проверка статуса лицензии

Если ключ уже активирован:
1. Откройте http://localhost:5678
2. Перейдите в **Settings** → **License**
3. Проверьте статус лицензии - должна быть активна

## Если ключ уже активирован

Если вы получили сообщение "этот код уже занят" или "key already activated":
- ✅ Это нормально - ключ уже привязан к вашему экземпляру n8n
- ✅ Лицензия должна быть активна
- ✅ Просто проверьте статус в **Settings** → **License**
- ❌ Не нужно активировать ключ повторно

## Сохранение ключа в config.env (опционально)

Хотя ключ уже активирован, можно сохранить его в `config.env` для справки:
```bash
cd /projects/brats
nano config.env
```

Добавьте:
```bash
N8N_LICENSE_KEY=bab52b80-51e4-4035-9a81-16c2b10637f0
```

Это не обязательно, но полезно для документации.

## Проверка лицензии

После ввода ключа проверьте статус лицензии:
- В n8n перейдите в **Settings** → **License**
- Должен отображаться статус лицензии

## Важно

- ⚠️ **Не коммитьте `config.env` в git** (он в `.gitignore`)
- Лицензионный ключ хранится только локально
- При перезапуске контейнера ключ загружается из `config.env`

## Получение нового ключа

Если нужен новый license key:
1. Перейдите на https://n8n.io/pricing
2. Выберите подходящий план
3. Получите новый ключ
4. Обновите `N8N_LICENSE_KEY` в `config.env`
