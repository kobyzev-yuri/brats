# Настройка первого пользователя в n8n

## Проблема: "wrong username or password"

Если вы получаете ошибку "wrong username or password" при попытке войти в n8n, это означает, что:

1. Учетная запись уже была создана ранее с другими данными
2. Или нужно создать первого пользователя через переменные окружения

## Решение 1: Использование переменных окружения (рекомендуется)

В новых версиях n8n можно создать первого пользователя автоматически через переменные окружения.

### Настройка в config.env

Отредактируйте `/projects/brats/config.env`:

```bash
# Первый пользователь n8n
N8N_USER_FIRST_NAME=Admin
N8N_USER_LAST_NAME=User
N8N_USER_EMAIL=admin@example.com
N8N_USER_PASSWORD=changeme
```

### Перезапуск n8n

```bash
cd /projects/brats/n8n
docker stop n8n-brats
docker rm n8n-brats
./start_n8n.sh
```

После перезапуска войдите используя:
- **Email**: `admin@example.com` (или тот, что указали в config.env)
- **Password**: `changeme` (или тот, что указали в config.env)

## Решение 2: Сброс данных n8n (если забыли пароль)

Если вы забыли пароль или хотите начать заново:

```bash
# Остановка n8n
docker stop n8n-brats
docker rm n8n-brats

# Удаление данных (⚠️ ВНИМАНИЕ: это удалит все workflows и настройки!)
rm -rf ~/.n8n

# Запуск заново
cd /projects/brats/n8n
./start_n8n.sh
```

После этого:
1. Откройте http://localhost:5678
2. Создайте новую учетную запись через веб-интерфейс
3. Или используйте переменные окружения из config.env

## Решение 3: Проверка существующих пользователей

Если данные уже есть в `~/.n8n/`, но вы не помните пароль:

1. Остановите n8n: `docker stop n8n-brats`
2. Удалите только пользовательские данные (сохранив workflows):
   ```bash
   # Бэкап workflows
   cp -r ~/.n8n/workflows ~/.n8n/workflows.backup
   
   # Удаление пользователей (но не workflows)
   rm -rf ~/.n8n/database.sqlite
   ```
3. Запустите n8n заново и создайте нового пользователя

## Важные замечания

- Переменные `N8N_USER_*` работают только при **первом запуске** n8n (когда база данных пуста)
- Если пользователь уже существует, эти переменные игнорируются
- Для сброса нужно удалить `~/.n8n/database.sqlite` или всю директорию `~/.n8n/`
- Email должен быть валидным форматом (не `admin@localhost`, используйте `admin@example.com`)

## Проверка текущих настроек

```bash
# Проверка переменных окружения в контейнере
docker exec n8n-brats env | grep N8N_USER

# Проверка логов на наличие ошибок
docker logs n8n-brats | grep -i "user\|error"
```
