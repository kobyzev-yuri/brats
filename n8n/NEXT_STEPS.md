# Следующие шаги после импорта workflow

## ✅ Что уже сделано

1. ✅ n8n установлен и запущен
2. ✅ Пользователь создан и лицензия активирована
3. ✅ Workflow импортирован

## 📋 Что нужно сделать дальше

### Шаг 1: Проверка и настройка узлов workflow

Откройте импортированный workflow и проверьте следующие узлы:

#### 1. Узел "KB Search"
- **URL**: должен быть `http://localhost:8001/api/kb/search`
- **Method**: POST
- **Body**: должен содержать параметры поиска

**Проверка KB Service:**
```bash
curl http://localhost:8001/health
```

Должен вернуться: `{"status": "healthy", ...}`

Если KB Service не запущен:
```bash
cd /projects/brats/kb-service
./start.sh
```

#### 2. Узел "Sales Agent Call"
- **URL**: `http://localhost:8000/api/chat` (или ваш URL агента продаж)
- Если агент еще не реализован:
  - Можно временно закомментировать этот узел
  - Или использовать mock endpoint для тестирования
  - Или оставить как есть - workflow будет работать, но агент не будет вызываться

#### 3. Узел "Webhook"
- После активации workflow будет показан URL webhook
- Пример: `http://localhost:5678/webhook/sales-agent-kb`
- Запомните этот URL для тестирования

### Шаг 2: Активация workflow (Publish)

В новом интерфейсе n8n workflow активируется через кнопку **Publish**:

1. В правом верхнем углу редактора workflow найдите кнопку **Publish**
2. Нажмите **Publish** - workflow будет опубликован и активирован
3. После публикации workflow станет активным и готов принимать запросы
4. URL webhook будет показан в узле Webhook или в настройках workflow

**Альтернативный способ:**
- Через меню (три точки **"..."**) → **Settings** → включите **Active** или нажмите **Publish**

### Шаг 3: Тестирование workflow

#### Тест через curl

```bash
curl -X POST http://localhost:5678/webhook/sales-agent-kb \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Расскажите о ценах на коттеджи",
    "channel": "telegram",
    "external_id": "test_user_123",
    "metadata": {
      "user_type": "end_buyer"
    }
  }'
```

#### Ожидаемый результат

1. ✅ Webhook получает запрос
2. ✅ Данные нормализуются
3. ✅ Выполняется поиск в KB (если KB Service запущен)
4. ✅ Результаты форматируются для промпта
5. ⚠️ Вызывается агент продаж (если настроен, иначе пропускается)
6. ✅ Возвращается ответ

### Шаг 4: Просмотр выполнения

1. В n8n перейдите в **Executions** (в левом меню)
2. Вы увидите историю выполнения workflow
3. Кликните на выполнение, чтобы увидеть детали каждого узла
4. Проверьте входные/выходные данные каждого узла

## 🔧 Устранение проблем

### Workflow не активируется
- Проверьте, что все узлы настроены правильно
- Убедитесь, что нет ошибок в узлах (красные индикаторы)
- Проверьте логи выполнения в **Executions**

### KB Search не работает
- Убедитесь, что KB Service запущен: `curl http://localhost:8001/health`
- Проверьте URL в узле "KB Search"
- Проверьте логи KB Service: `cd /projects/brats/kb-service && tail -f logs/*.log`

### Webhook не отвечает
- Убедитесь, что workflow активирован
- Проверьте правильность URL webhook (он показывается после активации)
- Проверьте логи n8n: `docker logs n8n-brats`

### Ошибки в узлах
- Откройте узел с ошибкой
- Проверьте входные данные
- Проверьте настройки узла
- Посмотрите сообщение об ошибке

## 🎯 Цель

После настройки и тестирования workflow должен:
1. Принимать запросы через webhook
2. Искать релевантную информацию в KB
3. Форматировать результаты для промпта
4. Вызывать агента продаж (когда будет готов)
5. Возвращать ответ клиенту
