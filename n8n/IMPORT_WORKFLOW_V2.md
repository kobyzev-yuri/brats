# Импорт workflow в новом интерфейсе n8n

## Способ 1: Через кнопку "New workflow"

1. В разделе **Workflows** нажмите кнопку **"New workflow"** (или **"+"** в правом верхнем углу)
2. Откроется редактор workflow
3. В правом верхнем углу редактора найдите меню (три точки **"..."** или иконка меню)
4. Выберите **"Import from File"** или **"Import"** → **"From File"**
5. Выберите файл: `/projects/brats/n8n/workflows/sales-agent-kb-integration.json`

## Способ 2: Через меню Workflows

1. В разделе **Workflows** найдите кнопку **"+"** или **"Add workflow"**
2. В выпадающем меню выберите **"Import from File"** или **"Import"**
3. Выберите файл: `/projects/brats/n8n/workflows/sales-agent-kb-integration.json`

## Способ 3: Перетаскивание файла (Drag & Drop)

1. Откройте редактор workflow (нажмите **"New workflow"**)
2. Перетащите файл `sales-agent-kb-integration.json` прямо в редактор
3. Workflow будет автоматически импортирован

## Способ 4: Через URL (если файл доступен по HTTP)

1. В редакторе workflow найдите меню (три точки)
2. Выберите **"Import from URL"**
3. Введите URL файла (если он размещен на сервере)

## Способ 5: Копирование JSON напрямую

1. Откройте файл `/projects/brats/n8n/workflows/sales-agent-kb-integration.json`
2. Скопируйте весь JSON
3. В редакторе workflow найдите меню (три точки)
4. Выберите **"Import from Clipboard"** или **"Paste"**
5. Вставьте JSON

## Если не видите опцию импорта

Попробуйте:
1. **Создать новый workflow** → затем в редакторе найти меню импорта
2. **Проверить права доступа** - возможно нужны права администратора
3. **Обновить страницу** (Ctrl+F5) - возможно кэш браузера
4. **Проверить версию n8n** - в новых версиях интерфейс может отличаться

## Альтернатива: Создание workflow вручную

Если импорт не работает, можно создать workflow вручную:
1. Нажмите **"New workflow"**
2. Добавьте узлы по описанию из `sales-agent-kb-integration.json`
3. Настройте каждый узел согласно документации в `N8N_INTEGRATION.md`

## Структура workflow для ручного создания

1. **Webhook** - триггер для получения сообщений
2. **Code** - нормализация входящих данных
3. **HTTP Request** - поиск в KB (`http://localhost:8001/api/kb/search`)
4. **Code** - форматирование результатов KB
5. **HTTP Request** - вызов агента продаж
6. **Code** - подготовка ответа
7. **Respond to Webhook** - возврат ответа

Подробное описание каждого узла см. в `N8N_INTEGRATION.md`.
