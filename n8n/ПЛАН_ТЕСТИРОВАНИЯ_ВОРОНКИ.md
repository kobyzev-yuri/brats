# План завершения тестирования воронки продаж

Цель: пройти воронку «как клиент» от первого сообщения до записи на просмотр и убедиться, что все узлы workflow и сервисы работают.

---

## Предварительные условия

- Все сервисы запущены (см. [NEXT_STEPS_AFTER_SERVICES_UP.md](NEXT_STEPS_AFTER_SERVICES_UP.md)):
  - **n8n** (5678), **kb-service** (8001), **amocrm-api** (8010), **funnel-api** (8011)
  - Чат с сайта: прокси/сайт на 8000 при необходимости
- Workflow **Sales Agent - KB Integration** импортирован из `workflows/sales-agent-kb-integration-localhost.json`, включён **Active**, в узле LLM ProxyAPI выбран credential.
- В `config.env`: AMOCRM_API_BASE_URL, AMOCRM_ACCESS_TOKEN, AMOCRM_SUBDOMAIN (для лидов и памяти диалога); DATABASE_URL (для amocrm-api и funnel-api).
- Миграции БД применены, в т.ч. `09_add_conversation_external_id.sql` (память диалога), `07_viewing_slots_and_templates.sql` (календарь). Слоты созданы: `curl -X POST "http://localhost:8011/api/viewing-slots/seed?days=7"`.

---

## Сценарий: прохождение воронки клиентом

Используйте один и тот же **external_id** во всех запросах (например `funnel-test-001`), чтобы работала память диалога.

### Шаг 1. Первый контакт (интерес, без телефона)

```bash
curl -X POST http://localhost:5678/webhook/sales-agent-kb \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Расскажите о ценах на коттеджи и условиях",
    "channel": "website",
    "external_id": "funnel-test-001",
    "metadata": {}
  }'
```

**Ожидание:** ответ по базе знаний, приглашение оставить контакт. В **Executions** — цепочка Resolve Conversation → DLP → KB Search → LLM → Respond to Webhook; при наличии external_id — Resolve вернёт conversation_id, после ответа — Save Messages API сохранит реплики.

---

### Шаг 2. Оставляем контакт (телефон + имя)

```bash
curl -X POST http://localhost:5678/webhook/sales-agent-kb \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Меня зовут Иван, перезвоните +79161234567",
    "channel": "website",
    "external_id": "funnel-test-001",
    "metadata": {}
  }'
```

**Ожидание:** ответ в духе «Спасибо, мы свяжемся». В AmoCRM — тестовый лид с префиксом [BRATS-TEST] и тегом brats_test. В **Executions** дополнительно проходят Maybe Create Lead → Create Lead If Phone → AmoCRM Create Lead.

---

### Шаг 3. Запрос свободных слотов (опционально)

Можно отправить сообщение: «Хочу записаться на просмотр, когда свободно?» — бот должен ответить с учётом слотов из Get Viewing Slots (если funnel-api поднят и слоты засеяны).

---

### Шаг 4. Запись на конкретный слот

```bash
curl -X POST http://localhost:5678/webhook/sales-agent-kb \
  -H "Content-Type: application/json" \
  -d '{
    "message": "24 февраля 10:00–11:00 запишите",
    "channel": "website",
    "external_id": "funnel-test-001",
    "metadata": {}
  }'
```

**Ожидание:** ответ в духе «Вы записаны на просмотр 24 февраля с 10:00 до 11:00». В БД должна появиться запись в `viewing_slots` со статусом `booked`. Проверка:

```bash
# В PostgreSQL
psql -U postgres -d brats -c "SELECT id, conversation_id, slot_start, slot_end, status, contact_name, contact_phone FROM viewing_slots WHERE status = 'booked' ORDER BY created_at DESC LIMIT 5;"
```

В **Executions** для этого запроса должны пройти узлы Parse Slot From Message → Book Slot If Parsed → Book Viewing Slot API (при успешном парсинге даты/времени).

## Чеклист завершения тестирования

| # | Проверка | Где смотреть |
|---|----------|----------------|
| 1 | Ответ бота на общий вопрос | Ответ curl / чат на сайте |
| 2 | Память диалога (один external_id) | Повторный запрос с тем же external_id — ответ с учётом контекста |
| 3 | Сохранение реплик в БД | amocrm-api: таблицы conversations, messages (conversation_id не null) |
| 4 | Создание лида при указании телефона | AmoCRM: сделка [BRATS-TEST], тег brats_test |
| 5 | Запись в календарь при фразе «дата время запишите» | PostgreSQL: viewing_slots со status = 'booked' |
| 6 | Нет красных узлов в Executions | n8n → Executions → последние запуски |
