# Проверка выполнения workflow

## ✅ Workflow запущен успешно!

Вы получили ответ: `{"message":"Workflow was started"}` - это означает, что:
- ✅ Webhook работает корректно
- ✅ Workflow начал выполняться
- ✅ Запрос принят и обрабатывается

## 📊 Как проверить результаты выполнения

### Шаг 1: Откройте Executions

1. В n8n перейдите в раздел **Executions** (в левом меню)
2. Вы увидите список всех выполнений workflow
3. Найдите последнее выполнение (самое свежее вверху списка)

### Шаг 2: Откройте выполнение

1. Кликните на выполнение, чтобы открыть детали
2. Вы увидите все узлы workflow и их статус:
   - ✅ **Зеленый** - узел выполнен успешно
   - ⚠️ **Желтый** - узел выполняется
   - ❌ **Красный** - узел завершился с ошибкой

### Шаг 3: Проверьте каждый узел

#### 1. Узел "Webhook"
- **Input**: должен содержать ваше сообщение
- **Output**: должен содержать данные запроса

#### 2. Узел "Normalize Input"
- **Input**: данные из Webhook
- **Output**: нормализованные данные с полями:
  - `message`: "Расскажите о ценах на коттеджи"
  - `channel`: "telegram"
  - `external_id`: "test_user_123"
  - `target_audience`: "end_buyer"

#### 3. Узел "KB Search"
- **Input**: нормализованные данные
- **Output**: результаты поиска из KB:
  ```json
  {
    "results": [
      {
        "id": 1,
        "content": "...",
        "similarity": 0.85,
        "metadata": {
          "category": "...",
          "target_audience": "..."
        }
      }
    ],
    "total": 5
  }
  ```

**Если KB Search вернул ошибку:**
- Проверьте, что KB Service запущен: `curl http://localhost:8001/health`
- Проверьте URL в узле: должен быть `http://localhost:8001/api/kb/search`
- Проверьте логи KB Service

#### 4. Узел "Format KB Context"
- **Input**: результаты KB Search
- **Output**: отформатированный контекст для промпта

#### 5. Узел "Sales Agent Call"
- Если агент продаж еще не реализован, этот узел может вернуть ошибку - это нормально
- Можно временно закомментировать этот узел или настроить mock endpoint

#### 6. Узел "Prepare Response"
- **Input**: ответ от агента (или данные из предыдущих узлов)
- **Output**: подготовленный ответ для клиента

#### 7. Узел "Respond to Webhook"
- **Input**: финальный ответ
- **Output**: ответ отправлен клиенту

## 🔍 Что проверить

### Успешное выполнение

Если все узлы зеленые:
- ✅ Workflow работает корректно
- ✅ KB Search нашел релевантные chunks
- ✅ Данные обработаны правильно

### Ошибки в узлах

Если какой-то узел красный:
1. Откройте узел с ошибкой
2. Посмотрите сообщение об ошибке
3. Проверьте входные данные узла
4. Проверьте настройки узла

### Частые проблемы

#### KB Search не работает
- **Проблема**: Узел возвращает ошибку 404 или 500
- **Решение**: 
  - Проверьте, что KB Service запущен
  - Проверьте URL: `http://localhost:8001/api/kb/search`
  - Проверьте, что в KB есть данные

#### Sales Agent Call не работает
- **Проблема**: Узел возвращает ошибку (агент не реализован)
- **Решение**: 
  - Это нормально, если агент еще не готов
  - Можно временно закомментировать этот узел
  - Или настроить mock endpoint для тестирования

#### Format KB Context не работает
- **Проблема**: Ошибка в JavaScript коде
- **Решение**: 
  - Проверьте входные данные узла
  - Проверьте синтаксис JavaScript кода
  - Убедитесь, что данные из KB Search корректны

