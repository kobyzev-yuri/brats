## Текущая ситуация

## Предлагаемые скрипты для sales-agent

### 1. Анализ и улучшение диалогов

#### 1.1. `analyze_conversations.py` - Анализ эффективности диалогов

**Назначение:** Анализ диалогов для выявления паттернов успеха/неудачи

**Что делает:**
- Анализирует завершенные диалоги из таблицы `conversations`
- Определяет успешные диалоги (привели к КП/сделке)
- Выявляет паттерны успешных сообщений
- Анализирует причины handoff
- Генерирует отчеты по эффективности

**Использование:**
```bash
python scripts/analyze_conversations.py --days 30 --output report.json
```

**Выходные данные:**
- Статистика по диалогам (количество, средняя длительность, конверсия)
- Топ успешных фраз/паттернов
- Причины handoff (частота, категории)
- Рекомендации по улучшению

**Статус:** ⏳ Рекомендуется создать

---

#### 1.2. `detect_handoff_patterns.py` - Обнаружение паттернов handoff

**Назначение:** Автоматическое обнаружение ситуаций, требующих передачи менеджеру

**Что делает:**
- Анализирует диалоги, которые закончились handoff
- Выявляет общие паттерны (фразы, темы, состояния FSM)
- Создает правила для автоматического определения необходимости handoff
- Обновляет конфигурацию handoff детектора

**Использование:**
```bash
python scripts/detect_handoff_patterns.py --days 60 --min-frequency 3
```

**Выходные данные:**
- Список паттернов, которые часто приводят к handoff
- Рекомендации по правилам handoff
- Обновленный конфиг для handoff детектора

**Статус:** ⏳ Рекомендуется создать

---

### 2. Экспорт и импорт данных

#### 2.1. `export_conversations.py` - Экспорт диалогов

**Назначение:** Экспорт диалогов для анализа, бэкапа или миграции

**Что делает:**
- Экспортирует диалоги из PostgreSQL в JSON/CSV
- Фильтрация по дате, статусу, каналу
- Включает метаданные и связи с amoCRM

**Использование:**
```bash
python scripts/export_conversations.py --from-date 2026-01-01 --format json --output conversations.json
```

**Статус:** ⏳ Рекомендуется создать

---

#### 2.2. `import_conversations.py` - Импорт диалогов

**Назначение:** Импорт диалогов из внешних источников или бэкапа

**Что делает:**
- Импортирует диалоги из JSON/CSV
- Валидация данных
- Создание записей в PostgreSQL
- Восстановление связей с amoCRM (если возможно)

**Использование:**
```bash
python scripts/import_conversations.py --file conversations.json --dry-run
```

**Статус:** ⏳ Рекомендуется создать

### 3. Тестирование и отладка

#### 3.1. `test_conversation_flow.py` - Тестирование FSM воронки

**Назначение:** Автоматическое тестирование состояний FSM и переходов

**Что делает:**
- Тестирует все состояния FSM (GREETING, QUALIFYING, PROPOSAL и т.д.)
- Проверяет корректность переходов между состояниями
- Тестирует обработку возражений
- Проверяет интеграцию с KB и proposal-generator

**Использование:**
```bash
python scripts/test_conversation_flow.py --test-all
python scripts/test_conversation_flow.py --test-state QUALIFYING
```

**Статус:** ⏳ Рекомендуется создать

---

#### 3.2. `simulate_conversation.py` - Симуляция диалога

**Назначение:** Симуляция диалога для тестирования без реального клиента

**Что делает:**
- Создает тестовый диалог
- Отправляет сообщения от имени "клиента"
- Проверяет ответы агента
- Валидирует переходы состояний

**Использование:**
```bash
python scripts/simulate_conversation.py --scenario objection_handling
```

**Статус:** ⏳ Рекомендуется создать

---

### 4. Мониторинг и аналитика

#### 4.1. `monitor_active_conversations.py` - Мониторинг активных диалогов

**Назначение:** Мониторинг активных диалогов и выявление проблем

**Что делает:**
- Отслеживает активные диалоги (без ответа > N минут)
- Выявляет "застрявшие" диалоги
- Отправляет уведомления о проблемах
- Генерирует метрики в реальном времени

**Использование:**
```bash
python scripts/monitor_active_conversations.py --check-interval 300 --alert-threshold 30
```

**Статус:** ⏳ Рекомендуется создать (можно запускать через cron)

---

#### 4.2. `generate_conversation_report.py` - Генерация отчетов

**Назначение:** Генерация отчетов по диалогам для менеджеров

**Что делает:**
- Анализирует диалоги за период
- Генерирует отчеты (HTML/PDF/JSON)
- Включает метрики: конверсия, средняя длительность, причины handoff
- Экспортирует в amoCRM как примечания

**Использование:**
```bash
python scripts/generate_conversation_report.py --period daily --format html --output report.html
```

**Статус:** ⏳ Рекомендуется создать

### 5. Обогащение KB из диалогов

#### 5.1. `extract_kb_from_conversations.py` - Извлечение знаний из диалогов

**Назначение:** Автоматическое извлечение полезной информации из диалогов для KB

**Что делает:**
- Анализирует успешные диалоги
- Извлекает эффективные ответы на вопросы
- Извлекает успешные паттерны обработки возражений
- Добавляет в KB через KB Service API

**Использование:**
```bash
python scripts/extract_kb_from_conversations.py --days 30 --min-success-rate 0.7
```

**Статус:** ⏳ Рекомендуется создать (аналогично скриптам в sales-analytic)

### 6. Миграция и обслуживание

#### 6.1. `migrate_conversations.py` - Миграция данных диалогов

**Назначение:** Миграция диалогов при изменении схемы БД

**Что делает:**
- Миграция данных между версиями схемы
- Обновление формата метаданных
- Валидация целостности данных
- Создание бэкапа перед миграцией

**Использование:**
```bash
python scripts/migrate_conversations.py --from-version 1.0 --to-version 2.0 --dry-run
```

**Статус:** ⏳ Создать при необходимости миграции

---

#### 6.2. `cleanup_old_conversations.py` - Очистка старых диалогов

**Назначение:** Архивация или удаление старых диалогов

**Что делает:**
- Находит диалоги старше N дней
- Архивирует их (экспорт + удаление из основной таблицы)
- Сохраняет статистику
- Очищает связанные данные (messages, proposals)

**Использование:**
```bash
python scripts/cleanup_old_conversations.py --older-than-days 365 --archive --dry-run
```

**Статус:** ⏳ Рекомендуется создать (для продакшена)

**Рекомендация:** Скрипты для сложной логики и анализа, n8n workflows для простых задач и интеграций.
