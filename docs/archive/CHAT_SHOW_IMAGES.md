# Показ релевантных картинок и документов агентом в чате

Структура KB и формат ответа описаны в **docs/KB_MEDIA_STRUCTURE.md**. Ниже — краткая схема.

Агент может показывать в ответе картинки и документы из двух источников:

1. **Ссылки с сайта / из БЗ** — URL уже есть в чанках (например в presentation_realtor.txt, price.txt: ссылки на innovatory-club.ru, Yandex.Disk и т.д.). Достаточно передать их в контекст LLM и отображать в чате.
2. **Картинки с диска** — браузер не имеет доступа к файловой системе. Нужно отдавать файлы по HTTP (статическая папка или API), а в БЗ хранить URL вида `http://localhost:8000/images/...` или `https://ваш-домен/images/...`.

---

## Что сделать по шагам

### 1. KB: чтобы в поиск попадали URL картинок

- Чанки уже могут содержать в **metadata** поле **images** (список `{url, description, alt}`) — при импорте по URL и сохранении блоков/галереи в админке.
- В ответе **POST /api/kb/search** в каждом элементе **results** есть **metadata**. Если в чанке есть `metadata.images`, эти URL можно использовать.

### 2. n8n: передать URL картинок в контекст LLM

В ноде **Format KB Context** при сборке `kbContext` для каждого результата из `kbResponse.results`:

- Проверять `r.metadata && r.metadata.images && Array.isArray(r.metadata.images)`.
- Для каждого изображения добавлять в текст контекста URL и при необходимости описание, например:  
  `Изображение: [описание] ${img.url}` или просто список URL.

В **системный промпт** добавить правило:

- «Если в блоке «База знаний» есть ссылки на фото (планировки, отделка, презентации) — включи 1–3 наиболее релевантные в конец ответа. Формат: отдельной строкой «Фото:» и затем ссылки с короткой подписью (что за фото).»

Тогда LLM будет возвращать в тексте ответа эти URL, и их можно будет отобразить.

### 3. Чат на сайте: отображать картинки в ответе

В `site-integration-example.html` в функции, которая выводит сообщение ассистента (например `appendMessage('assistant', reply)`):

- Либо парсить ответ: искать URL (например по регулярке для `https?://[^\s]+\.(jpg|jpeg|png|gif|webp)` или строку «Фото:» и ссылки после неё).
- Либо договориться о формате (например Markdown `![подпись](url)`) и рендерить разметку.
- Для каждого найденного URL вставлять в разметку сообщения, например:  
  `<img src="..." alt="..." class="chat-image" style="max-width:100%; border-radius:8px;" />`  
  или обёртку с лайтбоксом.

Так пользователь увидит релевантные картинки прямо в диалоге.

### 4. Картинки с диска

- Создать папку, например `site-integration/images` или `data/images`, и складывать туда файлы (фото отделок, планировок и т.д.).
- Раздавать их по HTTP:
  - в **serve_with_proxy.py** добавить обработку пути вида `/images/*` и отдавать файлы из этой папки (или использовать `StaticFiles`),  
  - либо в kb-service/другом сервисе — маршрут вида `GET /api/images/<id>` с отдачей файла с диска.
- В БЗ (в контенте или в metadata.images) хранить уже **URL**, например:  
  `http://localhost:8000/images/design_remont_1.jpg`  
  Дальше цепочка та же: n8n передаёт эти URL в контекст → LLM включает в ответ → чат отображает по ним картинки.

---

## Итог

| Источник              | Что нужно |
|-----------------------|-----------|
| Сайт / ссылки в БЗ    | Передавать URL из `metadata.images` и контента в контекст n8n, попросить LLM включать ссылки в ответ, парсить и показывать их в чате. |
| Картинки на диске     | Раздавать папку по HTTP (прокси или API), в БЗ хранить URL на этот сервер; дальше — так же, как для ссылок с сайта. |

После этого агент сможет в диалоге «показывать» релевантные картинки — и с сайта, и с диска (через их URL).
