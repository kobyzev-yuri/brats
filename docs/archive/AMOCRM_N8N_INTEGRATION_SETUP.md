# Создание внешней интеграции amoCRM для n8n

Пошаговая инструкция по созданию **внешней интеграции** в amoCRM и подключению её к n8n (в том числе для workflow создания лидов, примечаний, задач и синхронизации каталога).

---

## Общая схема

1. В n8n создаёте учётные данные amoCRM и **копируете URL перенаправления**.
2. В amoCRM в разделе **amoМаркет → Создать интеграцию** вставляете этот URL и сохраняете интеграцию.
3. В n8n завершаете OAuth: открываете ссылку авторизации, входите в amoCRM и разрешаете доступ.
4. После этого workflow n8n могут работать с API amoCRM (лиды, контакты, сделки, каталог и т.д.).

---

## Шаг 1: Получить URL перенаправления из n8n

Сначала n8n должен быть запущен и доступен по тому же адресу, который потом будет указан в интеграции (локально или по домену).

1. Откройте n8n (например, `http://localhost:5678` или ваш `N8N_URL` из `config.env`).
2. Перейдите в **Settings (Настройки)** → **Credentials** или откройте любой workflow.
3. Добавьте новые учётные данные:
   - Нажмите **Add Credential** (или в узле выберите тип **AmoCRM**).
   - Если в списке есть **AmoCRM** (встроенный или community node) — выберите его.
   - В форме учётных данных найдите поле **OAuth Redirect URL** / **Ссылка для перенаправления** — n8n покажет готовый URL, например:
     - Локально: `http://localhost:5678/rest/oauth2-credential/callback`
     - Или с вашим доменом: `https://your-n8n-domain.com/rest/oauth2-credential/callback`
4. **Скопируйте этот URL** — он понадобится в форме создания интеграции в amoCRM.

Если встроенного узла AmoCRM нет, можно использовать **HTTP Request** и **OAuth2 API** (см. [AMOCRM_API_SETUP.md](./AMOCRM_API_SETUP.md)); в этом случае callback URL будет от OAuth2 API ноды (n8n тоже его покажет при настройке credential).

---

## Шаг 2: Создать интеграцию в amoCRM

Форма соответствует разделу **amoМаркет → Создать интеграцию** (как на вашем скриншоте).

1. Войдите в аккаунт amoCRM (например, `innovatoryclub.amocrm.ru`).
2. В левом меню выберите **amoМаркет**.
3. Нажмите **Создать интеграцию** (или аналог для внешней интеграции).
4. Заполните поля:

| Поле в форме amoCRM | Что указать |
|---------------------|-------------|
| **Ссылка для перенаправления** | Вставьте **точную** ссылку из n8n (OAuth Redirect URL), например `http://localhost:5678/rest/oauth2-credential/callback` или ваш продакшен-URL n8n. |
| **Ссылка для хука об отключении (необязательно)** | Оставьте пустым или укажите URL, который вызывается при отключении интеграции (например, webhook n8n), если это нужно. |
| **Предоставить доступ** | Включите **Все** (или выберите нужные права: лиды, контакты, каталоги, примечания, задачи и т.д. — в зависимости от возможностей формы). |
| **Контроль дублей** / **Множественные источники** | По необходимости. |
| **Language / Язык** | Например, **Русский**. |
| **Название интеграции** | Например: `n8n` или `Нейропродажник (n8n)`. |
| **Описание** | Кратко: «Интеграция для автоматизации: лиды, примечания, задачи, каталог через n8n». |

5. Загрузите изображение **400×272 px** по желанию (можно пропустить, если не требуется).
6. Нажмите **Сохранить**.

После сохранения amoCRM покажет **Client ID** (ID интеграции) и **Client Secret** (секретный ключ). Они понадобятся в n8n.

---

## Шаг 3: Настроить учётные данные amoCRM в n8n

1. В n8n откройте те же учётные данные amoCRM (или создайте заново).
2. Заполните:
   - **Subdomain** — поддомен вашего аккаунта (например, `innovatoryclub` для `innovatoryclub.amocrm.ru`).
   - **Client ID** — из карточки интеграции в amoCRM.
   - **Client Secret** — из карточки интеграции в amoCRM.
   - **OAuth Redirect URL** — оставьте тот, что показывает n8n (он должен **совпадать** с тем, что указан в amoCRM в поле «Ссылка для перенаправления»).
3. Сохраните credential и нажмите **Sign in with amoCRM** (или аналог) — откроется окно авторизации amoCRM.
4. Войдите в amoCRM и разрешите доступ приложению. После успешного входа n8n получит токены и сохранит их в учётных данных.

Если используете **HTTP Request** + **OAuth2 API** (без встроенного AmoCRM node), подставьте в OAuth2 те же Client ID, Client Secret и Redirect URI и пройдите авторизацию так, как описано в [AMOCRM_API_SETUP.md](./AMOCRM_API_SETUP.md).

---

## Шаг 4: Использование в workflow

После успешной авторизации:

- В workflow можно использовать узлы **AmoCRM** (если установлены) с выбранными учётными данными.
- Либо узлы **HTTP Request** с заголовком `Authorization: Bearer {{ $credentials.amocrm.accessToken }}` и базовым URL `https://{{ $credentials.amocrm.subdomain }}.amocrm.ru/api/v4/...`.

Наши workflow (см. [n8n/workflows/README.md](../n8n/workflows/README.md)) ожидают переменные окружения или credentials:

- `AMOCRM_SUBDOMAIN` — поддомен.
- `AMOCRM_ACCESS_TOKEN` — токен доступа (при использовании HTTP Request и обновлении токена через отдельный flow или скрипт).

Либо один раз пройти OAuth через n8n (как в шаге 3) и далее использовать сохранённые учётные данные amoCRM в нодах.

---

## Важно

- **Ссылка для перенаправления** в amoCRM и **OAuth Redirect URL** в n8n должны совпадать **символ в символ** (включая `http`/`https`, порт и путь).
- Для продакшена используйте HTTPS и реальный домен n8n (например, `https://n8n.yourdomain.com/rest/oauth2-credential/callback`).
- Не храните Client Secret и токены в репозитории; используйте `config.env` (уже в `.gitignore`) или секреты окружения. См. [AMOCRM_API_SETUP.md](./AMOCRM_API_SETUP.md) и [n8n/CONFIG_SETUP.md](../n8n/CONFIG_SETUP.md).

## Краткий чек-лист

- [ ] n8n запущен, доступен по `N8N_URL`.
- [ ] В n8n созданы/открыты учётные данные amoCRM, скопирован **OAuth Redirect URL**.
- [ ] В amoCRM создана интеграция: в поле **Ссылка для перенаправления** вставлен этот URL.
- [ ] Сохранена интеграция, скопированы **Client ID** и **Client Secret**.
- [ ] В n8n в учётных данных указаны Subdomain, Client ID, Client Secret, выполнена кнопка **Sign in with amoCRM** и пройдена авторизация.
- [ ] В workflow используются учётные данные amoCRM (или переменные для HTTP Request).
