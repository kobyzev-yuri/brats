# Руководство: Исключение дубликатов при записи в Knowledge Base

## Задача

При записи смысловых блоков в Knowledge Base нужно проверять на дубликаты и не записывать повторно те же самые блоки.

## Проблема дубликатов

Дубликаты могут возникать когда:
1. Один и тот же блок присутствует на разных вкладках
2. Блок повторяется на разных страницах
3. Блок частично изменен, но суть та же
4. Блок присутствует в разных форматах (текст, список, таблица)

## Стратегия дедупликации

### Уровень 1: Точное совпадение

Проверка на полное совпадение содержимого блока.

### Уровень 2: Семантическое сходство

Проверка на смысловое сходство (блоки с одинаковым смыслом, но разным формулированием).

### Уровень 3: Структурное сходство

Проверка на структурное сходство (одинаковая структура, но разные данные).

## Алгоритм проверки дубликатов

### Шаг 1: Нормализация блока

Перед сравнением нормализовать блок:
- Удалить лишние пробелы
- Привести к нижнему регистру (для текста)
- Удалить знаки препинания (опционально)
- Удалить HTML-теги
- Нормализовать форматирование чисел

### Шаг 2: Генерация идентификатора блока

Создать уникальный идентификатор для каждого блока:

```python
def generate_block_id(block):
    """Генерирует идентификатор блока для сравнения"""
    
    # Для текстовых блоков - хэш от нормализованного текста
    if block['type'] == 'text':
        normalized = normalize_text(block['content'])
        return hashlib.md5(normalized.encode()).hexdigest()
    
    # Для списков - хэш от отсортированных элементов
    elif block['type'] == 'list':
        items = sorted([normalize_text(item) for item in block['items']])
        return hashlib.md5('|'.join(items).encode()).hexdigest()
    
    # Для цен - нормализованное значение
    elif block['type'] == 'price':
        normalized_price = normalize_price(block['value'])
        return f"price_{normalized_price}"
    
    # Для контактов - комбинация полей
    elif block['type'] == 'contacts':
        phone = normalize_phone(block.get('phone', ''))
        email = normalize_email(block.get('email', ''))
        return hashlib.md5(f"{phone}|{email}".encode()).hexdigest()
```

### Шаг 3: Проверка существования в KB

Перед записью проверить:
1. Существует ли блок с таким же ID?
2. Существует ли семантически похожий блок?
3. Является ли это обновлением существующего блока?

### Шаг 4: Решение о записи

- **Новый блок** → Записать в KB
- **Точный дубликат** → Пропустить запись
- **Семантически похожий** → Решить: обновить или пропустить
- **Обновление существующего** → Обновить запись

## Типы блоков и их идентификация

### 1. Заголовок типа отделки

**Идентификатор:** Комбинация типа отделки + вкладка

```python
block_id = f"title_{tab_name}_{block_type}"
# Пример: "title_STANDARD_standard"
```

**Проверка дубликата:**
- Если существует блок с таким же `tab_name` и `block_type` → дубликат

### 2. Описание

**Идентификатор:** Хэш от нормализованного текста + тип вкладки

```python
normalized_text = normalize_text(block['content'])
block_id = f"description_{tab_name}_{hashlib.md5(normalized_text.encode()).hexdigest()}"
```

**Проверка дубликата:**
- Если существует блок с таким же хэшем для той же вкладки → дубликат
- Если семантическое сходство > 90% → возможный дубликат

### 3. Что включает (список)

**Идентификатор:** Хэш от отсортированных элементов + тип вкладки

```python
items = sorted([normalize_text(item) for item in block['items']])
items_hash = hashlib.md5('|'.join(items).encode()).hexdigest()
block_id = f"includes_{tab_name}_{items_hash}"
```

**Проверка дубликата:**
- Если список элементов идентичен → дубликат
- Если > 80% элементов совпадают → возможный дубликат

### 4. Стоимость

**Идентификатор:** Нормализованная цена + тип вкладки

```python
normalized_price = normalize_price(block['value'])  # "8950000"
block_id = f"price_{tab_name}_{normalized_price}"
```

**Проверка дубликата:**
- Если цена идентична для той же вкладки → дубликат
- Если цена отличается → это обновление, не дубликат

### 5. Галерея изображений

**Идентификатор:** Список URL изображений + тип вкладки

```python
image_urls = sorted([img['url'] for img in block['images']])
urls_hash = hashlib.md5('|'.join(image_urls).encode()).hexdigest()
block_id = f"gallery_{tab_name}_{urls_hash}"
```

**Проверка дубликата:**
- Если все URL идентичны → дубликат
- Если > 80% URL совпадают → возможный дубликат

### 6. Контакты

**Идентификатор:** Комбинация контактных данных

```python
phone = normalize_phone(block.get('phone', ''))
email = normalize_email(block.get('email', ''))
address = normalize_text(block.get('address', ''))
contacts_hash = hashlib.md5(f"{phone}|{email}|{address}".encode()).hexdigest()
block_id = f"contacts_{contacts_hash}"
```

**Проверка дубликата:**
- Контакты обычно одинаковы для всех вкладок
- Если контакты идентичны → дубликат (записать один раз)

### 7. Преимущества

**Идентификатор:** Хэш от отсортированных элементов

```python
items = sorted([normalize_text(item) for item in block['items']])
items_hash = hashlib.md5('|'.join(items).encode()).hexdigest()
block_id = f"advantages_{items_hash}"
```

**Проверка дубликата:**
- Преимущества обычно одинаковы для всех вкладок
- Если список идентичен → дубликат (записать один раз)

## Алгоритм записи в KB

```python
def save_to_kb(block, kb_connection):
    """
    Сохраняет блок в KB с проверкой на дубликаты
    
    Args:
        block: Смысловой блок для сохранения
        kb_connection: Соединение с Knowledge Base
    """
    
    # Шаг 1: Генерация идентификатора
    block_id = generate_block_id(block)
    
    # Шаг 2: Проверка существования
    existing_block = kb_connection.get_block(block_id)
    
    if existing_block:
        # Точный дубликат
        if is_exact_duplicate(block, existing_block):
            print(f"Пропуск дубликата: {block_id}")
            return False
        
        # Семантически похожий
        similarity = calculate_semantic_similarity(block, existing_block)
        if similarity > 0.9:
            print(f"Пропуск семантически похожего блока: {block_id} (similarity: {similarity})")
            return False
        
        # Обновление существующего
        if should_update(block, existing_block):
            print(f"Обновление существующего блока: {block_id}")
            kb_connection.update_block(block_id, block)
            return True
    
    # Шаг 3: Запись нового блока
    print(f"Запись нового блока: {block_id}")
    kb_connection.save_block(block_id, block)
    return True
```

## Функции нормализации

```python
import re
import hashlib

def normalize_text(text):
    """Нормализует текст для сравнения"""
    # Привести к нижнему регистру
    text = text.lower()
    
    # Удалить лишние пробелы
    text = re.sub(r'\s+', ' ', text)
    
    # Удалить знаки препинания (опционально)
    # text = re.sub(r'[^\w\s]', '', text)
    
    # Удалить HTML-теги
    text = re.sub(r'<[^>]+>', '', text)
    
    return text.strip()

def normalize_price(price_str):
    """Нормализует цену для сравнения"""
    # Удалить все кроме цифр
    digits = re.sub(r'\D', '', price_str)
    return digits

def normalize_phone(phone_str):
    """Нормализует телефон для сравнения"""
    # Удалить все кроме цифр и +
    phone = re.sub(r'[^\d+]', '', phone_str)
    return phone

def normalize_email(email_str):
    """Нормализует email для сравнения"""
    return email_str.lower().strip()

def calculate_semantic_similarity(block1, block2):
    """Вычисляет семантическое сходство между блоками"""
    # Использовать embedding модели или простой алгоритм
    # Например, cosine similarity между embeddings
    
    # Упрощенная версия: сравнение по тексту
    text1 = normalize_text(block1.get('content', ''))
    text2 = normalize_text(block2.get('content', ''))
    
    # Использовать библиотеку для вычисления сходства
    # Например, difflib или sentence-transformers
    from difflib import SequenceMatcher
    return SequenceMatcher(None, text1, text2).ratio()
```

## Специальные случаи

### Случай 1: Контакты и преимущества

Эти блоки обычно одинаковы для всех вкладок. Стратегия:
- Записать один раз с `tab_name = "ALL"` или `tab_name = None`
- При последующих проверках всегда находить дубликат

### Случай 2: Обновление цены

Если цена изменилась для той же вкладки:
- Это не дубликат, а обновление
- Обновить существующую запись

### Случай 3: Частичное совпадение списков

Если список "Что включает" частично совпадает:
- Если > 80% элементов совпадают → возможный дубликат
- Если < 80% → это разные версии, записать обе
