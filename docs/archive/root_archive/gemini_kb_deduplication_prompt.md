# Промпт для Gemini: Дедупликация при записи в KB

## Полный промпт

```
КОНТЕКСТ: Запись смысловых блоков в Knowledge Base с исключением дубликатов.

ЗАДАЧА: При записи блоков в KB проверять на дубликаты и не записывать повторно те же самые блоки.

ПРАВИЛА ДЕДУПЛИКАЦИИ:

1. ПЕРЕД ЗАПИСЬЮ ВСЕГДА:
   - Сгенерируй уникальный идентификатор блока
   - Проверь существование блока с таким ID в KB
   - Определи тип: новый блок / точный дубликат / семантически похожий / обновление

2. ГЕНЕРАЦИЯ ИДЕНТИФИКАТОРОВ:

   Заголовок типа отделки:
   - Формат: "title_{tab_name}_{block_type}"
   - Пример: "title_STANDARD_standard"
   - Если существует блок с таким же tab_name и block_type → дубликат

   Описание:
   - Нормализуй текст (нижний регистр, удали пробелы)
   - Создай хэш от нормализованного текста
   - Формат: "description_{tab_name}_{hash}"
   - Если существует блок с таким же хэшем для той же вкладки → дубликат

   Что включает (список):
   - Отсортируй элементы списка
   - Создай хэш от отсортированных элементов
   - Формат: "includes_{tab_name}_{hash}"
   - Если список элементов идентичен → дубликат

   Стоимость:
   - Нормализуй цену (только цифры)
   - Формат: "price_{tab_name}_{normalized_price}"
   - Если цена идентична для той же вкладки → дубликат
   - Если цена отличается → это обновление, не дубликат

   Контакты:
   - Комбинация: телефон + email + адрес
   - Создай хэш от комбинации
   - Формат: "contacts_{hash}"
   - Контакты одинаковы для всех вкладок → записать ОДИН РАЗ
   - Если существует блок контактов → всегда дубликат

   Преимущества:
   - Отсортируй элементы списка
   - Создай хэш от отсортированных элементов
   - Формат: "advantages_{hash}"
   - Преимущества одинаковы для всех вкладок → записать ОДИН РАЗ
   - Если существует блок преимуществ → всегда дубликат

   Галерея:
   - Отсортируй URL изображений
   - Создай хэш от отсортированных URL
   - Формат: "gallery_{tab_name}_{hash}"
   - Если все URL идентичны → дубликат

3. ПРОВЕРКА ДУБЛИКАТОВ:

   Точный дубликат:
   - Блоки идентичны после нормализации
   - Действие: Пропустить запись

   Семантически похожий:
   - Семантическое сходство > 90%
   - Действие: Пропустить запись

   Обновление:
   - Цена изменилась для той же вкладки
   - Галерея обновилась
   - Действие: Обновить существующий блок

   Новый блок:
   - Блок не существует в KB
   - Действие: Записать в KB

4. НОРМАЛИЗАЦИЯ:

   Текст:
   - Привести к нижнему регистру
   - Удалить лишние пробелы
   - Удалить HTML-теги
   - Удалить знаки препинания (опционально)

   Цена:
   - Удалить все кроме цифр
   - Пример: "8 950 000 Р" → "8950000"

   Телефон:
   - Удалить все кроме цифр и +
   - Пример: "+7 (988) 199-89-98" → "+79881998998"

   Email:
   - Привести к нижнему регистру
   - Удалить пробелы

5. ЛОГИРОВАНИЕ:

   Всегда логируй действия:
   - "✅ Запись нового блока: {block_id}"
   - "⏭️ Пропуск точного дубликата: {block_id}"
   - "⏭️ Пропуск семантически похожего блока: {block_id} (similarity: {value})"
   - "🔄 Обновление существующего блока: {block_id}"

6. СПЕЦИАЛЬНЫЕ СЛУЧАИ:

   Контакты и преимущества:
   - Эти блоки одинаковы для всех вкладок
   - Записать ОДИН РАЗ с tab_name = "ALL" или без tab_name
   - При последующих проверках всегда находить дубликат

   Изменение цены:
   - Если цена изменилась для той же вкладки → это обновление
   - Обновить существующую запись, не создавать новую

   Частичное совпадение списков:
   - Если > 80% элементов совпадают → возможный дубликат
   - Если < 80% → это разные версии, записать обе

ВАЖНО:
- ВСЕГДА проверяй на дубликаты перед записью
- НЕ записывай повторно те же самые блоки
- Используй нормализацию для корректного сравнения
- Учитывай специальные случаи (контакты, преимущества)
- Логируй все действия
```

## Краткая версия промпта

```
При записи блоков в KB:

1. ПЕРЕД ЗАПИСЬЮ:
   - Сгенерируй ID блока
   - Проверь существование в KB
   - Если дубликат → пропусти запись

2. ИДЕНТИФИКАТОРЫ:
   - Заголовок: "title_{tab}_{type}"
   - Описание: "description_{tab}_{hash}"
   - Что включает: "includes_{tab}_{hash}"
   - Цена: "price_{tab}_{normalized}"
   - Контакты: "contacts_{hash}" (один раз)
   - Преимущества: "advantages_{hash}" (один раз)

3. СПЕЦИАЛЬНЫЕ СЛУЧАИ:
   - Контакты/преимущества → записать один раз
   - Изменение цены → обновление, не дубликат

4. ЛОГИРОВАНИЕ:
   - ✅ Запись нового блока
   - ⏭️ Пропуск дубликата
   - 🔄 Обновление блока
```

## Пример использования

### Запрос:
```
Запиши смысловые блоки со страницы STANDARD в KB.
```

### Действия Gemini:

```
1. Извлечь смысловые блоки со страницы
2. Для каждого блока:
   a. Сгенерировать ID
   b. Проверить существование в KB
   c. Если дубликат → пропустить
   d. Если новый → записать
   e. Зафиксировать в логе

Пример лога:
✅ Запись нового блока: description_STANDARD_a1b2c3d4
⏭️ Пропуск точного дубликата: contacts_e5f6g7h8
✅ Запись нового блока: includes_STANDARD_i9j0k1l2
⏭️ Пропуск семантически похожего блока: advantages_m3n4o5p6 (similarity: 0.95)
🔄 Обновление существующего блока: price_STANDARD_8950000
```

## Чек-лист

Перед записью каждого блока:

- [ ] Сгенерирован уникальный ID
- [ ] Проверено существование в KB
- [ ] Определен тип (новый/дубликат/обновление)
- [ ] Принято решение о записи
- [ ] Записано в KB (если нужно)
- [ ] Зафиксировано в логе
