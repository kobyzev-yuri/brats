# Исправление: Вторичная обработка не отображается

## Проблема
Кнопки и функционал вторичной обработки не отображаются в интерфейсе.

## Решение

### 1. Перезапустите Flask приложение

Если приложение уже запущено, остановите его (Ctrl+C) и запустите заново:

```bash
cd /projects/brats
python3 kb_admin_app.py
```

### 2. Очистите кэш браузера

- Нажмите `Ctrl+Shift+R` (или `Cmd+Shift+R` на Mac) для жесткой перезагрузки
- Или откройте страницу в режиме инкогнито

### 3. Проверьте наличие кнопок

**В списке блоков (`/blocks`):**
- Должна быть кнопка "Вторичная обработка" (синяя, с иконкой повтора) в разделе массовых операций
- У каждого блока должна быть кнопка с иконкой повтора в колонке "Действия"

**На странице деталей блока (`/blocks/<id>`):**
- Должна быть кнопка "Вторичная обработка" рядом с другими действиями

**В меню:**
- Должен быть пункт "Вторичная обработка" в боковом меню

### 4. Проверьте консоль браузера

Откройте Developer Tools (F12) и проверьте вкладку Console на наличие ошибок JavaScript.

### 5. Проверьте роуты

Откройте в браузере:
- `http://localhost:5000/reprocess` - должна открыться страница вторичной обработки
- `http://localhost:5000/reprocess?block_ids=1,2,3` - должна открыться страница с выбранными блоками

## Что было исправлено

1. ✅ Удален дубликат функции `reprocessBlocks` из `blocks_list.html` (функция теперь только в `base.html`)
2. ✅ Добавлены кнопки в интерфейс
3. ✅ Создан роут `/reprocess`
4. ✅ Создан API endpoint `/api/blocks/reprocess`

## Если проблема сохраняется

1. Проверьте логи Flask приложения на наличие ошибок
2. Убедитесь, что все файлы сохранены:
   ```bash
   ls -la templates/reprocess.html
   ls -la templates/blocks_list.html
   ls -la templates/base.html
   ```
3. Проверьте, что роуты зарегистрированы:
   ```bash
   python3 -c "from kb_admin_app import app; print([str(r) for r in app.url_map.iter_rules() if 'reprocess' in str(r)])"
   ```
