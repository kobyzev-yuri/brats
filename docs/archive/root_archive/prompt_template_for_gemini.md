# Шаблон промпта для Gemini при работе с вкладками

## Базовый промпт для Gemini

```
ВАЖНО: При обработке URL с хэш-фрагментом вида #!/tab/1063728081-X:

1. Определи номер вкладки из URL:
   - -1 = BLACK BOX
   - -2 = WHITE BOX  
   - -3 = STANDARD
   - -4 = DESIGN

2. Используй ТОЛЬКО браузерные инструменты (browser_navigate, browser_wait_for, browser_snapshot), 
   НЕ используй веб-поиск для парсинга.

3. После навигации подожди минимум 3 секунды для выполнения JavaScript.

4. Парси ТОЛЬКО контент активной вкладки, указанной в URL. 
   ИГНОРИРУЙ контент других вкладок, даже если он присутствует в DOM.

5. Проверь правильность вкладки по индикаторам:
   - STANDARD: содержит "Просторный, уютный дом", "более 300 домов"
   - BLACK BOX: содержит "черновой ремонт включает", "8 350 000 Р"
   - WHITE BOX: содержит "предчистовой ремонт включает", "8 950 000 Р"
   - DESIGN: содержит "дизайнерский ремонт включает", "Дизайн-проект"

6. Верни результат ТОЛЬКО для активной вкладки.
```

## Пример использования

### Запрос пользователя:
```
При вводе url https://innovatory-club.ru/katalog#!/tab/1063728081-3
```

### Что должна сделать Gemini:

```
Понял. URL указывает на вкладку STANDARD (номер 3).

Использую браузерные инструменты для корректного парсинга:

1. browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")
2. browser_wait_for(time: 3) - жду выполнения JavaScript
3. browser_snapshot() - проверяю активную вкладку
4. Парсю ТОЛЬКО контент вкладки STANDARD
5. Игнорирую контент BLACK BOX, WHITE BOX, DESIGN

Результат будет содержать только информацию о STANDARD (стандартный ремонт).
```

## Расширенный промпт с примерами

```
КОНТЕКСТ: Пользователь предоставляет URL вида:
https://innovatory-club.ru/katalog#!/tab/1063728081-X

ПРАВИЛА ОБРАБОТКИ:

1. Хэш-фрагмент #!/tab/1063728081-X указывает на конкретную вкладку, 
   которую видит клиент в браузере.

2. Клиент видит ТОЛЬКО одну вкладку за раз, не все сразу.

3. Используй browser_navigate → browser_wait_for(3 сек) → browser_snapshot

4. После получения снимка страницы:
   - Найди активную вкладку по индикаторам
   - Извлеки ТОЛЬКО её контент
   - Игнорируй остальные вкладки

5. Если в результате смешан контент разных вкладок - это ОШИБКА.

ПРИМЕРЫ:

URL: .../tab/1063728081-1
Ожидаемый результат: Только BLACK BOX (черновой ремонт)
НЕ включать: WHITE BOX, STANDARD, DESIGN

URL: .../tab/1063728081-3  
Ожидаемый результат: Только STANDARD (стандартный ремонт)
НЕ включать: BLACK BOX, WHITE BOX, DESIGN
```

## Краткая версия для быстрого использования

```
При парсинге URL с #!/tab/1063728081-X:
- Используй browser_navigate + browser_wait_for(3) + browser_snapshot
- Парси ТОЛЬКО контент вкладки X (1=BLACK BOX, 2=WHITE BOX, 3=STANDARD, 4=DESIGN)
- Игнорируй остальные вкладки
```
