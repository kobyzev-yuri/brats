# Строгий промпт для Gemini: Парсинг вкладок

## Полный промпт

```
КОНТЕКСТ: Парсинг страницы с вкладками для извлечения контента конкретной вкладки.

КРИТИЧЕСКАЯ ПРОБЛЕМА: При парсинге все вкладки смешиваются в один поток.
РЕШЕНИЕ: Строго ограничить парсинг только активной вкладкой и четко разделить на блоки.

СТРОГИЕ ПРАВИЛА:

1. ОПРЕДЕЛЕНИЕ АКТИВНОЙ ВКЛАДКИ:
   - URL содержит #!/tab/1063728081-X
   - X = 1 → ТОЛЬКО BLACK BOX
   - X = 2 → ТОЛЬКО WHITE BOX
   - X = 3 → ТОЛЬКО STANDARD
   - X = 4 → ТОЛЬКО DESIGN
   - Запомни активную вкладку и используй её для всех проверок

2. ИДЕНТИФИКАЦИЯ ИНДИКАТОРОВ:
   
   BLACK BOX:
   - Заголовок: "BLACK BOX (черновой ремонт)"
   - Цена: "8 350 000 Р"
   - Текст: "черновой ремонт включает"
   
   WHITE BOX:
   - Заголовок: "WHITE BOX (предчистовой ремонт)"
   - Цена: "8 950 000 Р"
   - Текст: "предчистовой ремонт включает"
   
   STANDARD:
   - Заголовок: "STANDARD (стандартный ремонт)"
   - Цена: "8'950'000 Р + ремонт-700'000 Р"
   - Текст: "стандартный ремонт включает"
   - Описание: "Просторный, уютный дом с качественным ремонтом"
   
   DESIGN:
   - Заголовок: "DESIGN (дизайнерский ремонт)"
   - Цена: "8'950'000 Р + дизайнерский ремонт-1'500'000 Р"
   - Текст: "дизайнерский ремонт включает"

3. СТРОГАЯ ФИЛЬТРАЦИЯ КОНТЕНТА:
   
   ИЗВЛЕКАТЬ ТОЛЬКО:
   - Элементы, содержащие индикаторы активной вкладки
   - Контент после заголовка активной вкладки
   - Список "Что включает" для активной вкладки
   - Стоимость активной вкладки
   - Галерею активной вкладки
   
   СТРОГО ИГНОРИРОВАТЬ:
   - Элементы с заголовками других вкладок
   - Описания других вкладок
   - "Что включает" других вкладок
   - Стоимость других вкладок
   - Галереи других вкладок

4. ПРОВЕРКА КАЖДОГО БЛОКА:
   
   Перед извлечением блока проверь:
   - ✅ Содержит ли индикаторы активной вкладки?
   - ✅ НЕ содержит ли индикаторы других вкладок?
   - ✅ Относится ли к активной вкладке?
   
   Если блок содержит индикаторы других вкладок → ОТБРОСЬ его!

5. РАЗДЕЛЕНИЕ НА СМЫСЛОВЫЕ БЛОКИ:
   
   Каждый блок = отдельная сущность с четкими границами:
   
   Блок 1: Заголовок
   - Тип: heading
   - Содержимое: "STANDARD (стандартный ремонт)"
   - Tab: STANDARD
   
   Блок 2: Описание
   - Тип: description
   - Содержимое: "Просторный, уютный дом..."
   - Tab: STANDARD
   
   Блок 3: Что включает
   - Тип: includes
   - Заголовок: "Что включает"
   - Элементы: [список]
   - Tab: STANDARD
   
   Блок 4: Стоимость
   - Тип: price
   - Значение: "8'950'000 Р + ремонт-700'000 Р"
   - Tab: STANDARD
   
   Блок 5: Галерея
   - Тип: gallery
   - Изображения: [список]
   - Tab: STANDARD

6. ФОРМАТ РЕЗУЛЬТАТА:
   
   {
     "tab": "STANDARD",
     "blocks": {
       "title": {
         "type": "heading",
         "content": "STANDARD (стандартный ремонт)",
         "tab": "STANDARD"
       },
       "description": {
         "type": "description",
         "content": "...",
         "tab": "STANDARD"
       },
       "includes": {
         "type": "includes",
         "title": "Что включает",
         "items": [...],
         "tab": "STANDARD"
       },
       "price": {
         "type": "price",
         "value": "...",
         "tab": "STANDARD"
       }
     }
   }

7. ПРОВЕРКА ПЕРЕД ЗАПИСЬЮ В KB:
   
   Для каждого блока проверь:
   - [ ] Блок содержит индикаторы активной вкладки
   - [ ] Блок НЕ содержит индикаторы других вкладок
   - [ ] Блок четко отделен от других
   - [ ] Блок помечен tab = активная вкладка
   
   Только после всех проверок → записать в KB

КРИТИЧЕСКИ ВАЖНО:
- Один URL = одна вкладка = один набор блоков
- НЕ смешивай контент разных вкладок
- НЕ парси все вкладки одновременно
- РАЗДЕЛЯЙ на четкие смысловые блоки
- ПРОВЕРЯЙ каждый блок перед записью
```

## Краткая версия промпта

```
СТРОГИЕ ПРАВИЛА ПАРСИНГА ВКЛАДКИ:

1. URL → определи активную вкладку (X = 1/2/3/4)

2. Найди индикаторы активной вкладки в snapshot

3. Извлеки ТОЛЬКО контент с этими индикаторами

4. Проверь каждый блок:
   - Содержит индикаторы активной вкладки? ✅
   - НЕ содержит индикаторы других вкладок? ✅

5. Раздели на смысловые блоки:
   - Заголовок
   - Описание
   - Что включает
   - Стоимость
   - Галерея

6. Запиши в KB только блоки активной вкладки

ВАЖНО:
- НЕ смешивай контент разных вкладок
- НЕ парси все вкладки одновременно
- РАЗДЕЛЯЙ на четкие блоки
- ПРОВЕРЯЙ перед записью
```

## Пример использования

### Запрос:
```
Проанализируй страницу https://innovatory-club.ru/katalog#!/tab/1063728081-3
и извлеки смысловые блоки ТОЛЬКО для вкладки STANDARD.
```

### Действия Gemini:

```
1. Определил активную вкладку: STANDARD (из URL #!/tab/1063728081-3)

2. browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")
3. browser_wait_for(time: 3)
4. browser_snapshot()

5. Нашел индикаторы STANDARD:
   - Заголовок: "STANDARD (стандартный ремонт)" ✅
   - Цена: "8'950'000 Р + ремонт-700'000 Р" ✅
   - Текст: "стандартный ремонт включает" ✅

6. Извлек ТОЛЬКО контент STANDARD:
   - Заголовок STANDARD ✅
   - Описание после заголовка STANDARD ✅
   - Список "стандартный ремонт включает" ✅
   - Цена "8'950'000 Р + ремонт-700'000 Р" ✅
   - Игнорировал BLACK BOX, WHITE BOX, DESIGN ❌

7. Разделил на смысловые блоки:
   - Блок 1: Заголовок (tab: STANDARD)
   - Блок 2: Описание (tab: STANDARD)
   - Блок 3: Что включает (tab: STANDARD)
   - Блок 4: Стоимость (tab: STANDARD)

8. Проверил каждый блок:
   - Все блоки содержат индикаторы STANDARD ✅
   - Все блоки НЕ содержат индикаторы других вкладок ✅

9. Записал в KB только блоки STANDARD
```

## Чек-лист

Перед записью в KB:

- [ ] Определена активная вкладка из URL
- [ ] Найдены индикаторы активной вкладки
- [ ] Извлечен ТОЛЬКО контент активной вкладки
- [ ] Игнорирован контент других вкладок
- [ ] Каждый блок проверен на отсутствие смешивания
- [ ] Контент разделен на четкие смысловые блоки
- [ ] Каждый блок помечен tab = активная вкладка
- [ ] В результате нет упоминаний других вкладок

## Резюме

**Главные правила:**

1. ✅ Один URL = одна вкладка = один набор блоков
2. ✅ Используй индикаторы для идентификации
3. ✅ Извлекай ТОЛЬКО контент активной вкладки
4. ✅ Проверяй каждый блок перед записью
5. ✅ Разделяй на четкие смысловые блоки
6. ✅ НЕ смешивай контент разных вкладок

**Критически важно:**
- Строго ограничить парсинг только активной вкладкой
- Четко разделить на смысловые блоки
- Проверять каждый блок на отсутствие смешивания
