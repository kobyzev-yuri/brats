# ФИНАЛЬНЫЕ ИНСТРУКЦИИ: Строгий парсинг вкладок

## Проблема

При парсинге страницы все вкладки смешиваются в один поток без:
- Ограничений на активную вкладку
- Смыслового разделения на блоки
- Проверок на смешивание контента

## Решение

Строгие правила парсинга с проверками на каждом этапе.

## Использование при каждом запросе

### Когда пользователь запрашивает анализ страницы с вкладкой:

```
ВАЖНО: URL указывает на вкладку STANDARD (номер 3).

СТРОГИЕ ПРАВИЛА ПАРСИНГА:

1. Определи активную вкладку: STANDARD (из URL #!/tab/1063728081-3)

2. Используй browser_navigate → wait_for(3) → snapshot

3. Найди индикаторы STANDARD в snapshot:
   - Заголовок: "STANDARD (стандартный ремонт)"
   - Цена: "8'950'000 Р + ремонт-700'000 Р"
   - Текст: "стандартный ремонт включает"

4. Извлеки ТОЛЬКО контент со индикаторами STANDARD

5. Проверь каждый блок:
   - Содержит "STANDARD" или "стандартный"? ✅
   - НЕ содержит "BLACK BOX", "WHITE BOX", "DESIGN"? ✅
   - Если содержит другие вкладки → ОТБРОСЬ

6. Раздели на смысловые блоки:
   - Блок 1: Заголовок (tab: STANDARD)
   - Блок 2: Описание (tab: STANDARD)
   - Блок 3: Что включает (tab: STANDARD)
   - Блок 4: Стоимость (tab: STANDARD)
   - Блок 5: Галерея (tab: STANDARD)

7. Запиши в KB только блоки STANDARD

РЕЗУЛЬТАТ ДОЛЖЕН БЫТЬ:
- Только контент STANDARD
- Четко разделен на блоки
- Каждый блок помечен tab = "STANDARD"
- Нет упоминаний других вкладок
```

## Индикаторы для каждой вкладки

### STANDARD (X=3):
- ✅ "STANDARD (стандартный ремонт)"
- ✅ "8'950'000 Р + ремонт-700'000 Р"
- ✅ "стандартный ремонт включает"
- ✅ "Просторный, уютный дом с качественным ремонтом"
- ❌ Игнорировать: "BLACK BOX", "WHITE BOX", "DESIGN", "черновой", "предчистовой", "дизайнерский"

### BLACK BOX (X=1):
- ✅ "BLACK BOX (черновой ремонт)"
- ✅ "8 350 000 Р"
- ✅ "черновой ремонт включает"
- ❌ Игнорировать: "WHITE BOX", "STANDARD", "DESIGN", "предчистовой", "стандартный", "дизайнерский"

### WHITE BOX (X=2):
- ✅ "WHITE BOX (предчистовой ремонт)"
- ✅ "8 950 000 Р" (без доплат)
- ✅ "предчистовой ремонт включает"
- ❌ Игнорировать: "BLACK BOX", "STANDARD", "DESIGN", "черновой", "стандартный", "дизайнерский"

### DESIGN (X=4):
- ✅ "DESIGN (дизайнерский ремонт)"
- ✅ "8'950'000 Р + дизайнерский ремонт-1'500'000 Р"
- ✅ "дизайнерский ремонт включает"
- ✅ "Дизайн-проект — выбор из 3 готовых вариантов"
- ❌ Игнорировать: "BLACK BOX", "WHITE BOX", "STANDARD", "черновой", "предчистовой", "стандартный"

## Алгоритм проверки блока

Для каждого извлеченного блока:

```
1. Содержит ли индикаторы активной вкладки?
   - Если НЕТ → ОТБРОСЬ блок

2. Содержит ли индикаторы других вкладок?
   - Если ДА → ОТБРОСЬ блок

3. Четко ли отделен от других блоков?
   - Если НЕТ → РАЗДЕЛИ на отдельные блоки

4. Помечен ли tab = активная вкладка?
   - Если НЕТ → ДОБАВЬ tab
```

## Чек-лист перед записью в KB

Для каждого блока:

- [ ] Определена активная вкладка из URL
- [ ] Найдены индикаторы активной вкладки
- [ ] Блок содержит индикаторы активной вкладки
- [ ] Блок НЕ содержит индикаторы других вкладок
- [ ] Блок четко отделен от других
- [ ] Блок помечен tab = активная вкладка
- [ ] Блок имеет тип (heading/description/includes/price/gallery)
- [ ] Блок готов к записи в KB

## Примеры правильного и неправильного

### ❌ НЕПРАВИЛЬНО (смешанный контент):

```
Результат:
"BLACK BOX стоит 8 350 000 Р, 
WHITE BOX стоит 8 950 000 Р, 
STANDARD стоит 8'950'000 Р + ремонт-700'000 Р"
```

**Проблема:** Смешан контент всех вкладок в один блок

### ✅ ПРАВИЛЬНО (только STANDARD, разделено на блоки):

```
Результат:
{
  "tab": "STANDARD",
  "blocks": {
    "title": "STANDARD (стандартный ремонт)",
    "description": "Просторный, уютный дом...",
    "includes": {"items": [...]},
    "price": "8'950'000 Р + ремонт-700'000 Р"
  }
}
```

**Правильно:** Только STANDARD, четко разделено на блоки

## Резюме

**Главные правила:**

1. ✅ Один URL = одна вкладка = один набор блоков
2. ✅ Используй индикаторы для идентификации
3. ✅ Извлекай ТОЛЬКО контент активной вкладки
4. ✅ Проверяй каждый блок на отсутствие смешивания
5. ✅ Разделяй на четкие смысловые блоки
6. ✅ Помечай каждый блок tab = активная вкладка

**Критически важно:**
- Строго ограничить парсинг только активной вкладкой
- Четко разделить на смысловые блоки
- Проверять каждый блок перед записью
- НЕ смешивать контент разных вкладок
