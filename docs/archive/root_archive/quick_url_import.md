# Краткая инструкция: Импорт по URL

## Ответ на вопрос

**Да, Gemini должна сама обрабатывать URL без парсера**, используя browser инструменты напрямую.

## Алгоритм

```
1. Получить URL от пользователя

2. Обработать URL НАПРЯМУЮ (без парсера):
   browser_navigate(URL)
   browser_wait_for(time: 3)
   browser_snapshot()

3. Определить активную вкладку из URL (если есть хэш)

4. Извлечь смысловые блоки:
   - Только контент активной вкладки
   - Разделить на блоки: заголовок, описание, что включает, стоимость, галерея

5. Обработать через библиотекаря:
   - Нормализовать, структурировать, добавить метаданные

6. Записать в KB с проверкой на дубликаты
```

## Промпт для Gemini

```
ИМПОРТ ПО URL:

1. Обработай URL НАПРЯМУЮ (без парсера):
   browser_navigate(URL) → wait_for(3) → snapshot()

2. Определи активную вкладку из URL

3. Извлеки смысловые блоки:
   - Только контент активной вкладки
   - Раздели на блоки

4. Обработай через библиотекаря

5. Запиши в KB с проверкой на дубликаты

ВАЖНО:
- Gemini сама обрабатывает URL, не нужен парсер
- Извлекай только активную вкладку
- Разделяй на блоки
- Проверяй на дубликаты
```

## Сравнение

**❌ Старый подход:**
URL → Парсер → Текст → Библиотекарь → Gemini → KB

**✅ Новый подход:**
URL → Gemini (browser) → Блоки → Библиотекарь → KB

## Преимущества

- ✅ Видит активную вкладку
- ✅ Выполняет JavaScript
- ✅ Извлекает только нужный контент
- ✅ Сохраняет структуру
