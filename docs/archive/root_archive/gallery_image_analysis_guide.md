# Руководство по анализу галереи изображений на вкладке

## Задача

Проанализировать каждое изображение в галерее на вкладке, получить его URL и составить описание через Gemini Vision.

## Структура галереи

На странице есть галерея с:
- Кнопками навигации: "Предыдущий слайд" / "Следующий слайд"
- Индикатором текущего слайда: "X из Y" (например, "4 из 5")
- Изображениями, которые меняются при клике на кнопки

## Пошаговый алгоритм

### Шаг 1: Определение количества изображений

```javascript
// После browser_snapshot() найти индикатор галереи
// Искать паттерн: "X из Y" или "X of Y"
// Извлечь число Y - это общее количество изображений
```

**Пример:**
- Найден текст: "4 из 5" → всего 5 изображений
- Найден текст: "1 of 3" → всего 3 изображения

### Шаг 2: Получение URL всех изображений

#### Вариант A: Извлечение из DOM (если все изображения загружены)

```javascript
// После browser_snapshot() найти все элементы с role="img"
// Извлечь атрибут src или data-src для каждого изображения
// Фильтровать только изображения из галереи (не логотипы, иконки и т.д.)
```

#### Вариант B: Навигация по галерее (если изображения загружаются динамически)

```javascript
// 1. Начать с первого изображения
browser_snapshot() // Получить текущее изображение

// 2. Извлечь URL текущего изображения
// Найти элемент img в галерее, получить src

// 3. Перейти к следующему изображению
browser_click(element: "Следующий слайд", ref: "ref-ax17sgne27h")
browser_wait_for(time: 1) // Ждать загрузки нового изображения
browser_snapshot() // Получить новое изображение

// 4. Повторить для всех изображений
```

### Шаг 3: Анализ каждого изображения через Gemini Vision

Для каждого URL изображения:

```javascript
// Использовать Gemini Vision API или аналогичный инструмент
// Передать URL изображения и запросить описание

prompt = "Опиши это изображение детально. Что на нем изображено? 
Какие детали интерьера/экстерьера видны? Какой стиль? 
Какие материалы использованы?"
```

### Шаг 4: Сохранение результатов

Структура данных для каждого изображения:

```json
{
  "index": 1,
  "url": "https://thb.tildacdn.com/tild3661-3936-4762-b934-373132633739/-/empty/PHOTO-2025-04-14-14-.jpg",
  "description": "Интерьер гостиной в стиле минимализм. Светлые стены, современная мебель...",
  "type": "interior" // или "exterior", "plan", "detail" и т.д.
}
```

## Полный скрипт анализа

```javascript
// 1. Навигация на страницу с нужной вкладкой
browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")
browser_wait_for(time: 3)

// 2. Получение снимка страницы
browser_snapshot()

// 3. Определение количества изображений
// Найти индикатор "X из Y" или "X of Y"
total_images = extract_total_from_indicator() // например, 5

// 4. Список для хранения результатов
image_results = []

// 5. Цикл по всем изображениям
for (i = 1; i <= total_images; i++) {
    // Получить текущее изображение
    browser_snapshot()
    
    // Извлечь URL текущего изображения
    current_image_url = extract_image_url_from_snapshot()
    
    // Получить скриншот изображения для анализа
    screenshot = browser_take_screenshot(element: "current image")
    
    // Проанализировать через Gemini Vision
    description = analyze_image_with_gemini_vision(screenshot, current_image_url)
    
    // Сохранить результат
    image_results.push({
        index: i,
        url: current_image_url,
        description: description
    })
    
    // Если не последнее изображение - перейти к следующему
    if (i < total_images) {
        browser_click(element: "Следующий слайд", ref: "ref-ax17sgne27h")
        browser_wait_for(time: 1) // Ждать загрузки
    }
}

// 6. Вернуть все результаты
return image_results
```

## Альтернативный подход: Извлечение всех URL сразу

```javascript

// 2. Найти все элементы изображений в галерее

// 3. Извлечь URL всех изображений

// 4. Для каждого URL:

    // Загрузить изображение

    // Сохранить результат
    results.push({url: url, description: description})

```

## Использование Gemini Vision API

### Пример запроса для анализа изображения

```python
import google.generativeai as genai

# Настройка API
genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel('gemini-pro-vision')

# Анализ изображения
response = model.generate_content([
    "Опиши детально это изображение. Что на нем изображено? "
    "Какие детали интерьера/экстерьера видны? Какой стиль оформления? "
    "Какие материалы использованы? Какое настроение создает изображение?",
    image_url  # или загруженное изображение
])

description = response.text
```

## Проблемы и решения

### Проблема 1: Изображения загружаются динамически

**Решение:** Использовать навигацию по галерее (Вариант B)

### Проблема 2: Нельзя определить количество изображений

**Решение:** 
- Кликать "Следующий слайд" до тех пор, пока кнопка не станет неактивной
- Или искать максимальное число в индикаторе при прокрутке

### Проблема 3: Изображения в lazy loading

**Решение:**
- Прокрутить галерею, чтобы загрузить все изображения
- Или использовать data-src вместо src

### Проблема 4: Нужно анализировать только изображения галереи

**Решение:**
- Фильтровать по контейнеру галереи
- Исключать логотипы, иконки (по размеру, позиции, классам)

## Пример полного промпта для Gemini

```
Задача: Проанализировать все изображения в галерее на вкладке STANDARD.

Шаги:
1. Навигация: browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")
2. Ожидание: browser_wait_for(time: 3)
3. Снимок: browser_snapshot()
4. Определить количество изображений из индикатора "X из Y"
5. Для каждого изображения:
   a. Извлечь URL текущего изображения
   b. Сделать скриншот изображения: browser_take_screenshot()
   c. Проанализировать через Gemini Vision с промптом описания
   d. Сохранить URL и описание
   e. Если не последнее - кликнуть "Следующий слайд" и подождать 1 сек
6. Вернуть список всех изображений с URL и описаниями

Важно: Анализировать только изображения из галереи вкладки, не логотипы и иконки.
```

## Рекомендации

1. **Всегда ждите загрузки** после переключения слайда (1-2 секунды)
2. **Проверяйте индикатор** "X из Y" для контроля прогресса
3. **Сохраняйте URL** каждого изображения для последующего использования
4. **Используйте детальные промпты** для Gemini Vision для лучших описаний
5. **Группируйте результаты** по типу (интерьер, экстерьер, план и т.д.)
