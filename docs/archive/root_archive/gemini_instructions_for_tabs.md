# Инструкции для Gemini: Парсинг страниц с вкладками

## Важная информация для обработки URL с вкладками

Когда пользователь предоставляет URL вида:
```
https://innovatory-club.ru/katalog#!/tab/1063728081-3
```

**КРИТИЧЕСКИЙ МОМЕНТ**: Хэш-фрагмент `#!/tab/1063728081-X` указывает на конкретную вкладку, которую видит клиент.

## Правила обработки

### 1. Определение активной вкладки из URL

Извлеките номер вкладки из хэш-фрагмента:
- `#!/tab/1063728081-1` → **BLACK BOX** (черновой ремонт)
- `#!/tab/1063728081-2` → **WHITE BOX** (предчистовой ремонт)
- `#!/tab/1063728081-3` → **STANDARD** (стандартный ремонт)
- `#!/tab/1063728081-4` → **DESIGN** (дизайнерский ремонт)

### 2. Парсинг ТОЛЬКО активной вкладки

**НЕ ПАРСЬТЕ ВСЕ ВКЛАДКИ ОДНОВРЕМЕННО!**

Когда пользователь запрашивает информацию по URL с конкретной вкладкой:
- ✅ Парсите ТОЛЬКО контент активной вкладки
- ✅ Игнорируйте контент других вкладок
- ✅ Убедитесь, что JavaScript переключил вкладку перед парсингом

### 3. Использование браузерных инструментов

Для страниц с динамическим контентом ВСЕГДА используйте браузерные инструменты:

```javascript
// Шаг 1: Навигация на страницу с нужным хэшем
browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")

// Шаг 2: Ожидание выполнения JavaScript (минимум 2-3 секунды)
browser_wait_for(time: 3)

// Шаг 3: Проверка активной вкладки
browser_snapshot() // Проверьте, что активна правильная вкладка

// Шаг 4: При необходимости - явное переключение
// Если вкладка не переключилась автоматически:
browser_click(element: "STANDARD tab", ref: "ref-4i8bs6ks1zr")

// Шаг 5: Финальный снимок для парсинга
browser_snapshot()
```

### 4. Идентификация контента активной вкладки

После получения снимка страницы, ищите следующие индикаторы:

**BLACK BOX (вкладка 1):**
- Текст: "черновой ремонт включает"
- Цена: "8 350 000 Р"
- Кнопка: "Коттедж с черновой отделкой"

**WHITE BOX (вкладка 2):**
- Текст: "предчистовой ремонт включает"
- Цена: "8 950 000 Р"
- Кнопка: "Коттедж с предчистовой отделкой"

**STANDARD (вкладка 3):**
- Текст: "Просторный, уютный дом с качественным ремонтом"
- Текст: "более 300 домов"
- Текст: "минималистичная классика"
- Кнопка: "Коттедж со стандартным ремонтом"

**DESIGN (вкладка 4):**
- Текст: "дизайнерский ремонт включает"
- Текст: "Дизайн-проект — выбор из 3 готовых вариантов"
- Цена: "8'950'000 Р + дизайнерский ремонт-1'500'000 Р"
- Кнопка: "Коттедж с дизайнерским ремонтом"

### 5. Фильтрация контента

При парсинге страницы:

1. **Найдите активную вкладку** по индикаторам выше
2. **Извлеките ТОЛЬКО контент этой вкладки**:
   - Описание
   - Что включает
   - Стоимость
   - Кнопки резервирования
3. **ИГНОРИРУЙТЕ** контент других вкладок, даже если он присутствует в DOM

### 6. Пример промпта для пользователя

Когда пользователь запрашивает информацию, используйте такой подход:

```
Пользователь: "При вводе url https://innovatory-club.ru/katalog#!/tab/1063728081-3"

Ответ Gemini:
1. Определяю, что это вкладка STANDARD (номер 3)
2. Использую browser_navigate с этим URL
3. Жду выполнения JavaScript (browser_wait_for)
4. Проверяю, что активна вкладка STANDARD
5. Парсю ТОЛЬКО контент вкладки STANDARD
6. Игнорирую контент BLACK BOX, WHITE BOX, DESIGN
```

## Чек-лист для проверки

Перед возвратом результата пользователю убедитесь:

- [ ] Определен номер вкладки из URL
- [ ] Использованы браузерные инструменты (не веб-поиск)
- [ ] Выполнено ожидание JavaScript (минимум 2-3 секунды)
- [ ] Проверена активная вкладка
- [ ] Извлечен ТОЛЬКО контент активной вкладки
- [ ] Игнорирован контент других вкладок
- [ ] Результат соответствует запрошенной вкладке

## Примеры правильного и неправильного поведения

### ❌ НЕПРАВИЛЬНО:
```
URL: https://innovatory-club.ru/katalog#!/tab/1063728081-3

Результат: 
"BLACK BOX стоит 8 350 000 Р, WHITE BOX стоит 8 950 000 Р, 
STANDARD стоит 8 950 000 Р, DESIGN стоит 10 450 000 Р..."
```
**Проблема**: Смешан контент всех вкладок

### ✅ ПРАВИЛЬНО:
```
URL: https://innovatory-club.ru/katalog#!/tab/1063728081-3

Результат:
"STANDARD (стандартный ремонт):
- Описание: Просторный, уютный дом с качественным ремонтом...
- Что включает: [только для STANDARD]
- Стоимость: 8 950 000 Р"
```
**Правильно**: Только контент вкладки STANDARD

## Технические детали

### Почему веб-поиск не работает

Веб-поиск парсит статический HTML до выполнения JavaScript:
- JavaScript переключает вкладки на основе хэша URL
- Это происходит ПОСЛЕ загрузки HTML
- Поэтому веб-поиск всегда видит первую вкладку

### Почему браузерные инструменты работают

Браузерные инструменты:
- Выполняют JavaScript
- Дожидаются обновления DOM
- Получают актуальное состояние страницы
- Могут взаимодействовать с элементами

## Резюме
