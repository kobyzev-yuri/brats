# Промпт для Gemini: Импорт по URL

## Полный промпт

```
КОНТЕКСТ: Импорт страницы по URL в Knowledge Base.

ВАЖНО: Gemini должна сама обрабатывать URL напрямую через browser инструменты,
БЕЗ использования внешнего парсера.

АЛГОРИТМ ИМПОРТА:

1. ПОЛУЧИТЬ URL от пользователя
   Пример: "Импортируй https://innovatory-club.ru/katalog#!/tab/1063728081-3"

2. ОБРАБОТАТЬ URL НАПРЯМУЮ (НЕ использовать парсер):
   - browser_navigate(URL)
   - browser_wait_for(time: 3) - ждать выполнения JavaScript
   - browser_snapshot() - получить актуальное состояние страницы

3. ОПРЕДЕЛИТЬ активную вкладку из URL:
   - Если URL содержит #!/tab/1063728081-X
   - X = 1 → BLACK BOX
   - X = 2 → WHITE BOX
   - X = 3 → STANDARD
   - X = 4 → DESIGN

4. ИЗВЛЕЧЬ смысловые блоки из snapshot:
   - Использовать индикаторы активной вкладки
   - Извлечь ТОЛЬКО контент активной вкладки
   - Разделить на блоки:
     * Заголовок
     * Описание
     * Что включает
     * Стоимость
     * Галерея
   - Проверить каждый блок на отсутствие смешивания

5. ОБРАБОТАТЬ блоки через "библиотекаря":
   - Нормализовать текст
   - Структурировать данные
   - Добавить метаданные:
     * source_url: URL страницы
     * tab: активная вкладка
     * extracted_at: время извлечения
     * processed_by: "librarian"

6. ЗАПИСАТЬ в KB с проверкой на дубликаты:
   - Сгенерировать ID для каждого блока
   - Проверить существование в KB
   - Записать только новые блоки (не дубликаты)

ЗАПРЕЩЕНО:
- ❌ Использовать внешний парсер для извлечения текста
- ❌ Парсить статический HTML без JavaScript
- ❌ Смешивать контент разных вкладок
- ❌ Записывать дубликаты

ОБЯЗАТЕЛЬНО:
- ✅ Использовать browser инструменты напрямую
- ✅ Ждать выполнения JavaScript
- ✅ Извлекать только контент активной вкладки
- ✅ Разделять на четкие смысловые блоки
- ✅ Проверять на дубликаты перед записью
```

## Краткая версия

```
ИМПОРТ ПО URL:

1. Получи URL от пользователя

2. Обработай URL НАПРЯМУЮ (без парсера):
   browser_navigate(URL) → wait_for(3) → snapshot()

3. Определи активную вкладку из URL

4. Извлеки смысловые блоки:
   - Только контент активной вкладки
   - Раздели на блоки: заголовок, описание, что включает, стоимость, галерея

5. Обработай через библиотекаря:
   - Нормализуй, структурируй, добавь метаданные

6. Запиши в KB с проверкой на дубликаты

ВАЖНО:
- Gemini сама обрабатывает URL, не нужен парсер
- Извлекай только активную вкладку
- Разделяй на блоки
- Проверяй на дубликаты
```

## Пример использования

### Запрос:
```
Импортируй страницу https://innovatory-club.ru/katalog#!/tab/1063728081-3
```

### Действия Gemini:

```
1. Получил URL: https://innovatory-club.ru/katalog#!/tab/1063728081-3

2. Определил активную вкладку: STANDARD (из хэша)

3. Обработал URL напрямую (БЕЗ парсера):
   browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")
   browser_wait_for(time: 3)
   browser_snapshot()

4. Извлек смысловые блоки из snapshot:
   - Заголовок: "STANDARD (стандартный ремонт)"
   - Описание: "Просторный, уютный дом..."
   - Что включает: [список]
   - Стоимость: "8'950'000 Р + ремонт-700'000 Р"
   - Галерея: [изображения]

5. Обработал через библиотекаря:
   - Нормализовал текст
   - Структурировал данные
   - Добавил метаданные (source_url, tab: STANDARD)

6. Записал в KB:
   - Проверил на дубликаты
   - Записал только новые блоки
   - Пометил tab = "STANDARD"

Результат: Импортировано 5 блоков для вкладки STANDARD
```

## Чек-лист

Перед импортом:

- [ ] Получен URL от пользователя
- [ ] Определена активная вкладка (если есть хэш)
- [ ] Использованы browser инструменты (не парсер)
- [ ] Выполнено ожидание JavaScript (3 сек)
- [ ] Извлечен только контент активной вкладки
- [ ] Контент разделен на смысловые блоки
- [ ] Блоки обработаны через библиотекаря
- [ ] Проверены на дубликаты
- [ ] Записаны в KB только новые блоки

## Резюме

**Ответ:** Да, Gemini должна сама обрабатывать URL без парсера, используя browser инструменты напрямую.

**Преимущества:**
- Видит активную вкладку
- Выполняет JavaScript
- Извлекает только нужный контент
- Сохраняет структуру

**Не нужен:**
- Внешний парсер
- Промежуточная обработка HTML
- Статический парсинг
