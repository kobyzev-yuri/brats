# Строгие правила парсинга вкладок

## Критическая проблема

При парсинге страницы все вкладки парсятся одновременно, без учета активной вкладки из URL. Это приводит к смешиванию контента всех вкладок в один поток.

## Строгие правила для Gemini

### ПРАВИЛО 1: Определение активной вкладки из URL

```
URL: https://innovatory-club.ru/katalog#!/tab/1063728081-X

X = 1 → ТОЛЬКО BLACK BOX
X = 2 → ТОЛЬКО WHITE BOX  
X = 3 → ТОЛЬКО STANDARD
X = 4 → ТОЛЬКО DESIGN

КРИТИЧЕСКИ ВАЖНО: Парси ТОЛЬКО контент этой вкладки!
```

### ПРАВИЛО 2: Идентификация контента активной вкладки

После browser_snapshot() найди индикаторы активной вкладки:

**BLACK BOX (X=1):**
- Заголовок: "BLACK BOX (черновой ремонт)"
- Цена: "8 350 000 Р"
- Текст: "черновой ремонт включает"
- Кнопка: "Коттедж с черновой отделкой"

**WHITE BOX (X=2):**
- Заголовок: "WHITE BOX (предчистовой ремонт)"
- Цена: "8 950 000 Р"
- Текст: "предчистовой ремонт включает"
- Кнопка: "Коттедж с предчистовой отделкой"

**STANDARD (X=3):**
- Заголовок: "STANDARD (стандартный ремонт)"
- Цена: "8'950'000 Р + ремонт-700'000 Р"
- Текст: "Просторный, уютный дом с качественным ремонтом"
- Текст: "более 300 домов"
- Текст: "стандартный ремонт включает"
- Кнопка: "Коттедж со стандартным ремонтом"

**DESIGN (X=4):**
- Заголовок: "DESIGN (дизайнерский ремонт)"
- Цена: "8'950'000 Р + дизайнерский ремонт-1'500'000 Р"
- Текст: "дизайнерский ремонт включает"
- Текст: "Дизайн-проект — выбор из 3 готовых вариантов"
- Кнопка: "Коттедж с дизайнерским ремонтом"

### ПРАВИЛО 3: Строгая фильтрация контента

**ИЗВЛЕКАТЬ ТОЛЬКО:**
1. Заголовок активной вкладки (BLACK BOX / WHITE BOX / STANDARD / DESIGN)
2. Описание активной вкладки
3. "Что включает" активной вкладки
4. Стоимость активной вкладки
5. Галерея активной вкладки

**СТРОГО ИГНОРИРОВАТЬ:**
- ❌ Заголовки других вкладок
- ❌ Описания других вкладок
- ❌ "Что включает" других вкладок
- ❌ Стоимость других вкладок
- ❌ Галереи других вкладок

### ПРАВИЛО 4: Разделение на смысловые блоки

Каждый блок должен быть четко отделен:

```json
{
  "tab": "STANDARD",
  "blocks": {
    "title": {
      "type": "heading",
      "content": "STANDARD (стандартный ремонт)",
      "tab": "STANDARD"
    },
    "description": {
      "type": "text",
      "content": "Просторный, уютный дом с качественным ремонтом...",
      "tab": "STANDARD"
    },
    "includes": {
      "type": "list",
      "title": "Что включает",
      "items": [
        "Чистовая отделка — стены выровнены...",
        "Напольное покрытие — уложен ламинат..."
      ],
      "tab": "STANDARD"
    },
    "price": {
      "type": "price",
      "value": "8'950'000 Р + ремонт-700'000 Р",
      "tab": "STANDARD"
    },
    "gallery": {
      "type": "gallery",
      "images": [...],
      "tab": "STANDARD"
    }
  }
}
```

### ПРАВИЛО 5: Проверка перед записью в KB

Перед записью каждого блока проверь:

1. **Принадлежность к вкладке:**
   - Блок содержит индикаторы активной вкладки?
   - Блок НЕ содержит индикаторы других вкладок?

2. **Отсутствие смешивания:**
   - В блоке нет упоминаний других типов отделки?
   - Цена соответствует активной вкладке?
   - Описание соответствует активной вкладке?

3. **Четкое разделение:**
   - Блок имеет четкие границы?
   - Блок не смешан с другими блоками?

## Алгоритм строгого парсинга

```
1. browser_navigate("URL с хэшем #!/tab/1063728081-X")
2. browser_wait_for(time: 3)
3. browser_snapshot()

4. ОПРЕДЕЛИТЬ активную вкладку:
   - Извлечь X из URL
   - Найти индикаторы активной вкладки в snapshot
   - ПОДТВЕРДИТЬ, что активна правильная вкладка

5. ИЗВЛЕЧЬ ТОЛЬКО контент активной вкладки:
   - Найти заголовок активной вкладки
   - Найти описание после заголовка активной вкладки
   - Найти "Что включает" для активной вкладки
   - Найти стоимость активной вкладки
   - Найти галерею активной вкладки

6. ПРОВЕРИТЬ каждый блок:
   - Содержит ли индикаторы активной вкладки?
   - НЕ содержит ли индикаторы других вкладок?
   - Если содержит другие вкладки → ОТБРОСИТЬ блок

7. РАЗДЕЛИТЬ на смысловые блоки:
   - Каждый блок = отдельная сущность
   - Каждый блок имеет четкие границы
   - Каждый блок помечен tab = активная вкладка

8. ЗАПИСАТЬ в KB только блоки активной вкладки
```

## Примеры правильного и неправильного парсинга

### ❌ НЕПРАВИЛЬНО (смешанный контент):

```
URL: #!/tab/1063728081-3 (STANDARD)

Результат:
"BLACK BOX стоит 8 350 000 Р, 
WHITE BOX стоит 8 950 000 Р, 
STANDARD стоит 8'950'000 Р + ремонт-700'000 Р,
DESIGN стоит 8'950'000 Р + дизайнерский ремонт-1'500'000 Р"
```

**Проблема:** Смешан контент всех вкладок

### ✅ ПРАВИЛЬНО (только STANDARD):

```
URL: #!/tab/1063728081-3 (STANDARD)

Результат:
{
  "tab": "STANDARD",
  "blocks": {
    "title": "STANDARD (стандартный ремонт)",
    "description": "Просторный, уютный дом с качественным ремонтом...",
    "includes": {
      "title": "Что включает",
      "items": [
        "Чистовая отделка — стены выровнены...",
        "Напольное покрытие — уложен ламинат..."
      ]
    },
    "price": "8'950'000 Р + ремонт-700'000 Р"
  }
}
```

**Правильно:** Только контент STANDARD, четко разделен на блоки

## Промпт для Gemini

```
СТРОГИЕ ПРАВИЛА ПАРСИНГА ВКЛАДОК:

1. URL указывает на конкретную вкладку:
   - #!/tab/1063728081-1 → ТОЛЬКО BLACK BOX
   - #!/tab/1063728081-2 → ТОЛЬКО WHITE BOX
   - #!/tab/1063728081-3 → ТОЛЬКО STANDARD
   - #!/tab/1063728081-4 → ТОЛЬКО DESIGN

2. ПАРСИ ТОЛЬКО активную вкладку:
   - Найди индикаторы активной вкладки
   - Извлеки ТОЛЬКО контент с этими индикаторами
   - ИГНОРИРУЙ контент других вкладок

3. РАЗДЕЛИ на смысловые блоки:
   - Заголовок
   - Описание
   - Что включает
   - Стоимость
   - Галерея
   - Каждый блок = отдельная сущность

4. ПРОВЕРЬ перед записью:
   - Блок содержит индикаторы активной вкладки?
   - Блок НЕ содержит индикаторы других вкладок?
   - Блок четко отделен от других?

5. ЗАПИШИ в KB только блоки активной вкладки

КРИТИЧЕСКИ ВАЖНО:
- НЕ смешивай контент разных вкладок
- НЕ парси все вкладки одновременно
- НЕ создавай один большой блок со всем контентом
- ПАРСИ ТОЛЬКО активную вкладку
- РАЗДЕЛЯЙ на четкие смысловые блоки
```

## Чек-лист проверки

Перед записью в KB проверь:

- [ ] Определена активная вкладка из URL
- [ ] Найдены индикаторы активной вкладки в snapshot
- [ ] Извлечен ТОЛЬКО контент активной вкладки
- [ ] Игнорирован контент других вкладок
- [ ] Контент разделен на четкие смысловые блоки
- [ ] Каждый блок помечен tab = активная вкладка
- [ ] Каждый блок проверен на отсутствие смешивания
- [ ] В результате нет упоминаний других вкладок

## Резюме

**Главные правила:**

1. ✅ Парси ТОЛЬКО активную вкладку из URL
2. ✅ Используй индикаторы для идентификации контента
3. ✅ Разделяй на четкие смысловые блоки
4. ✅ Проверяй каждый блок перед записью
5. ✅ НЕ смешивай контент разных вкладок

**Критически важно:**
- Один URL = одна вкладка = один набор блоков
- Каждый блок четко отделен и помечен
- Нет смешивания контента разных вкладок
