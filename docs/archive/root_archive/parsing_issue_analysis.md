# Проблема парсинга страниц с динамическим контентом

## Описание проблемы

При парсинге страницы `https://innovatory-club.ru/katalog#!/tab/1063728081-3` через веб-поиск возвращается один и тот же контент для всех вариантов URL (с разными последними цифрами: 1, 2, 3), хотя на самом деле на странице должны отображаться разные блоки информации:

- `#!/tab/1063728081-1` → BLACK BOX (черновой ремонт)
- `#!/tab/1063728081-2` → WHITE BOX (предчистовой ремонт)  
- `#!/tab/1063728081-3` → STANDARD (стандартный ремонт)
- `#!/tab/1063728081-4` → DESIGN (дизайнерский ремонт)

## Причина проблемы

1. **Веб-поиск парсит статический HTML** до того, как JavaScript успевает выполниться
2. **JavaScript переключает вкладки** на основе хэш-фрагмента URL (`#!/tab/1063728081-3`)
3. **Время выполнения**: JavaScript требует времени для:
   - Загрузки скриптов
   - Обработки хэш-фрагмента
   - Переключения вкладки
   - Обновления DOM

## Решение

### Вариант 1: Использовать браузерные инструменты (рекомендуется)

Вместо веб-поиска использовать браузерные инструменты, которые:
- Дожидаются выполнения JavaScript
- Могут взаимодействовать с элементами страницы
- Получают актуальное состояние DOM после выполнения скриптов

**Пример использования:**
```javascript
// 1. Навигация на страницу с нужным хэшем
browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")

// 2. Ожидание загрузки JavaScript
browser_wait_for(time: 2)

// 3. Получение снимка страницы после выполнения JS
browser_snapshot()

// 4. При необходимости - клик на нужную вкладку для гарантии
browser_click(element: "STANDARD tab", ref: "ref-4i8bs6ks1zr")
```

### Вариант 2: Явное переключение вкладки

Если браузерные инструменты недоступны, можно:
1. Загрузить страницу
2. Вручную кликнуть на нужную вкладку
3. Подождать обновления контента
4. Затем парсить страницу

### Вариант 3: Парсинг через API (если доступно)

Если сайт предоставляет API для получения данных о разных типах ремонта, использовать его вместо парсинга HTML.

## Проверка правильности парсинга

После парсинга нужно проверить, что получен правильный контент:

- **BLACK BOX**: должен содержать текст "черновой ремонт включает", "8 350 000 Р"
- **WHITE BOX**: должен содержать текст "предчистовой ремонт включает", "8 950 000 Р"
- **STANDARD**: должен содержать текст "Просторный, уютный дом с качественным ремонтом", "более 300 домов"
- **DESIGN**: должен содержать текст "дизайнерский ремонт включает", "8'950'000 Р + дизайнерский ремонт-1'500'000 Р"

## Рекомендации

1. **Для страниц с динамическим контентом** всегда использовать браузерные инструменты
2. **Добавлять задержку** после загрузки страницы для выполнения JavaScript
3. **Проверять активную вкладку** перед парсингом контента
4. **Использовать индикаторы** (например, aria-selected, классы active) для определения активной вкладки
