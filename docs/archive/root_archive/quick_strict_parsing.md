# Краткая инструкция: Строгий парсинг вкладок

## Правило для Gemini

```
СТРОГИЙ ПАРСИНГ ВКЛАДКИ:

1. URL → определи активную вкладку (X = 1/2/3/4)

2. Найди индикаторы активной вкладки:
   - STANDARD: "STANDARD (стандартный ремонт)", "8'950'000 Р + ремонт-700'000 Р"
   - BLACK BOX: "BLACK BOX (черновой ремонт)", "8 350 000 Р"
   - WHITE BOX: "WHITE BOX (предчистовой ремонт)", "8 950 000 Р"
   - DESIGN: "DESIGN (дизайнерский ремонт)", "8'950'000 Р + дизайнерский ремонт-1'500'000 Р"

3. Извлеки ТОЛЬКО контент с индикаторами активной вкладки

4. Проверь каждый блок:
   - Содержит индикаторы активной вкладки? ✅
   - НЕ содержит индикаторы других вкладок? ✅
   - Если содержит другие → ОТБРОСЬ

5. Раздели на смысловые блоки:
   - Заголовок
   - Описание
   - Что включает
   - Стоимость
   - Галерея

6. Запиши в KB только блоки активной вкладки

ВАЖНО:
- НЕ смешивай контент разных вкладок
- НЕ парси все вкладки одновременно
- РАЗДЕЛЯЙ на четкие блоки
- ПРОВЕРЯЙ перед записью
```

## Индикаторы для проверки

**STANDARD:**
- ✅ "STANDARD (стандартный ремонт)"
- ✅ "8'950'000 Р + ремонт-700'000 Р"
- ✅ "стандартный ремонт включает"
- ✅ "Просторный, уютный дом"
- ❌ "BLACK BOX", "WHITE BOX", "DESIGN"
- ❌ "черновой", "предчистовой", "дизайнерский"

**BLACK BOX:**
- ✅ "BLACK BOX (черновой ремонт)"
- ✅ "8 350 000 Р"
- ✅ "черновой ремонт включает"
- ❌ "WHITE BOX", "STANDARD", "DESIGN"

## Формат результата

```json
{
  "tab": "STANDARD",
  "blocks": {
    "title": {"type": "heading", "content": "...", "tab": "STANDARD"},
    "description": {"type": "description", "content": "...", "tab": "STANDARD"},
    "includes": {"type": "includes", "items": [...], "tab": "STANDARD"},
    "price": {"type": "price", "value": "...", "tab": "STANDARD"}
  }
}
```

## Чек-лист

- [ ] Определена активная вкладка из URL
- [ ] Найдены индикаторы активной вкладки
- [ ] Извлечен ТОЛЬКО контент активной вкладки
- [ ] Каждый блок проверен на отсутствие других вкладок
- [ ] Контент разделен на четкие блоки
- [ ] Каждый блок помечен tab = активная вкладка
