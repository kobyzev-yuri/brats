### Основные компоненты системы

1. **Нейроаналитик** (`analytics-service/`)
   - Анализ поведения на сайте
   - Обнаружение горячих лидов
   - Генерация отчётов

2. **Нейропродажник** (`sales-agent/`)
   - Диалоги с лидами
   - FSM воронки продаж
   - Обработка возражений

3. **Proposal Generator** (`proposal-generator/`)
   - Генерация коммерческих предложений
   - Модификация КП в переговорах
   - Создание договоров купли-продажи

4. **Procurement Agent** (`procurement-agent/`)
   - Закупки для клиентов-организаций

5. **Оркестратор** (`n8n/`)
   - Единый workflow «Sales Agent - KB Integration»: webhook → нормализация → DLP (обезличивание) → память диалога (amocrm-api resolve/save) → KB Search → LLM → ответ; при телефоне — лид в AmoCRM; при фразе «дата время запишите» — запись в календарь (funnel-api).
   - Workflow: `n8n/workflows/sales-agent-kb-integration-localhost.json`. Описание узлов и потоков: `n8n/BUSINESS_PROCESSES.md`. План прохождения воронки клиентом: `n8n/ПЛАН_ТЕСТИРОВАНИЯ_ВОРОНКИ.md`.
   - Интеграции: amoCRM (лиды, память диалога), kb-service (поиск, DLP), funnel-api (календарь просмотров).

6. **База данных** (`db-init/`, PostgreSQL)
   - PostgreSQL: основные данные
   - Redis: кэш и очереди

### Каналы и маршруты обработки заявок

**Внешние каналы:**
- Telegram‑бот (личные чаты) и бот в группах/каналах (мониторинг триггерных сообщений).
- Avito (новые диалоги и заявки по объявлениям).
- Сайт/веб‑чат/формы (лиды с лендингов и посадочных страниц).

**Единый маршрут через n8n:**
- Все события с каналов приходят в n8n (webhook/API).
- n8n нормализует данные и связывает событие с контактом/сделкой в amoCRM.
- Далее используется один из двух маршрутов:
  - **Маршрут через AI (нейропродажник)** — дефолтный для типовых лидов.
  - **Прямой маршрут к менеджеру** — для VIP/сложных/особых кейсов.

**Правила маршрутизации:**
- Реализуются в виде If/Switch‑узлов в n8n:
  - проверка канала (`telegram`, `avito`, `site` и т.п.);
  - сегмента клиента (VIP/обычный, новые/текущие);
  - текста сообщения (ключевые слова, крупные бюджеты);
  - времени (ночь/день, рабочее/нерабочее время).
- Результат маршрутизации:
  - вызов `sales-agent` и продолжение диалога через выбранный канал;
  - или прямое создание/обновление сделки и задачи менеджеру в amoCRM.

### n8n Chat Hub и внутренняя работа команды

**Роль n8n Chat Hub** (см. [Introducing Chat Hub](https://blog.n8n.io/introducing-chat-hub/)):
- Единая точка входа для менеджеров, маркетологов и руководителей.
- Интерфейс “как чат”, за которым стоят workflow‑агенты n8n.
- Централизованное управление доступами и токенами (на уровне организации).

**Планируемые агенты Chat Hub:**
- **“Нейропродажник (помощник)”**
  - резюмирует диалоги с лидами;
  - подсказывает, что делать по конкретной сделке;
  - генерирует черновики ответов и КП.
- **“Аналитик по каналам”**
  - отвечает на вопросы про Telegram/Avito/сайт (количество лидов, конверсия, CPA/ROI);
  - строит on‑demand отчёты и поясняет аномалии.
- **“Интегратор с amoCRM”**
  - позволяет находить/редактировать сделки и контакты по естественным запросам.

**Роли:**
- *Builders* — разрабатывают workflow в `n8n-flows/`, настраивают интеграции с сервисами.
- *Chat users* — работают только через Chat Hub, без доступа к техническим деталям.

## План реализации интеграции каналов (черновик для декомпозиции задач)

### Этап 1. Базовый сквозной сценарий (MVP)

- Настроить в n8n:
  - webhook для Telegram‑бота (в тестовом режиме);
  - простой flow: входящее сообщение → нормализация → логирование в БД (или mock‑сервис).
- В `sales-agent`:
  - создать endpoint приёма сообщений по каналу/внешнему ID;
  - реализовать минимальный FSM (GREETING → QUALIFYING → HANDOFF) без сложной логики.
- В amoCRM:
  - поднять рабочую интеграцию (по `docs/AMOCRM_API_SETUP.md`);
  - сделать первый flow “Telegram → sales-agent → amoCRM”.

### Этап 2. Общий контур маршрутизации (AI vs менеджер)

- Добавить в n8n отдельный flow‑контейнер для маршрутизации:
  - вход: нормализованное событие;
  - выход: “вызвать sales-agent” или “идти напрямую в amoCRM”.
- Прописать первые правила:
  - по типу канала;
  - по наличию тега VIP/ключевых слов (“крупный бюджет”, “опт”, “срочно” и т.п.).
- Доработать `sales-agent`, чтобы он возвращал флаг/решение о handoff (нужен ли менеджер).

### Этап 3. Интеграция Avito и унификация логики каналов

- Добавить в n8n flow для Avito:
  - подключение API/вебхука;
  - нормализация структуры сообщения;
  - передача в общий маршрутизатор.
- В amoCRM:
  - завести поле “Источник лидов” с возможностью фильтровать Avito/Telegram/сайт;
  - убедиться, что все новые лиды помечаются корректным источником.

### Этап 4. Подготовка внутренних агентов в Chat Hub

- Поднять n8n Chat Hub (см. [документацию](https://docs.n8n.io/advanced-ai/chat-hub/)):
  - настроить базовые роли и доступы;
  - привязать необходимые API‑ключи на уровне организации.
- Создать минимум 2 workflow‑агента:
  - “Нейропродажник (помощник менеджера)”;
  - “Аналитик по каналам”.
- Описать в `n8n-flows/README.md`:
  - какие flow относятся к каналам;
  - какие — к Chat Hub агентам;
  - как эти части связаны с сервисами `sales-agent` и `analytics-service`.

## Особенности работы

### Разделение данных: PostgreSQL vs amoCRM vs n8n

- **amoCRM**: используется для лидов и сделок до продажи
- **PostgreSQL**: хранит диалоги, КП, договоры, каталог объектов, базу знаний (KB), варианты скидок и продаж
- **n8n**: workflows для оркестрации компонентов и интеграций

### Жизненный цикл клиента

```
Лид → Квалификация → КП → Договор → Сделка завершена
```

1. Лид появляется (сайт, Telegram, Avito)
2. Нейропродажник квалифицирует (использует KB из PostgreSQL через векторный поиск)
3. Генерируется КП
4. Создаётся договор
5. После оплаты → сделка завершена в amoCRM

### Ключевые процессы

- **Генерация КП**: автоматическая на основе потребностей клиента
- **Модификация КП**: версионирование в процессе переговоров
- **Создание договора**: на основе финализированного КП
- **Поиск в KB**: через PostgreSQL pgvector (векторный поиск по таблице knowledge_base)
- **Поиск объектов**: через PostgreSQL pgvector (векторный поиск по таблице products)
- **Деанонимизация**: ненавязчивое получение контактов от анонимных посетителей
- **Обогащение KB**: автоматическое добавление новых возражений и вопросов
- **Аналитика**: онлайн и оффлайн отчёты для улучшения конверсии
