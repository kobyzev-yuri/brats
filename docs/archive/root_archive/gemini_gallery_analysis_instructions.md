# Инструкции для Gemini: Анализ галереи изображений на вкладке

## Задача

Когда пользователь запрашивает анализ галереи изображений на вкладке, нужно:
1. Найти все изображения в галерее
2. Получить URL каждого изображения
3. Проанализировать каждое изображение через Gemini Vision
4. Составить описание для каждого изображения

## Алгоритм действий

### Шаг 1: Навигация и подготовка

```
1. browser_navigate("URL с нужной вкладкой")
2. browser_wait_for(time: 3) - ждать выполнения JavaScript
3. browser_snapshot() - получить снимок страницы
```

### Шаг 2: Определение количества изображений

Найти в snapshot индикатор галереи:
- Искать текст вида "X из Y" или "X of Y"
- Извлечь число Y - это общее количество изображений
- Пример: "4 из 5" → всего 5 изображений

### Шаг 3: Извлечение URL изображений

**Вариант A: Если все изображения в DOM**

```
1. Найти все элементы с role="img" в контейнере галереи
2. Извлечь атрибут src или data-src
3. Фильтровать только изображения галереи (не логотипы, иконки)
```

**Вариант B: Если изображения загружаются динамически (рекомендуется)**

```
Для каждого изображения (от 1 до total_images):
1. browser_snapshot() - получить текущее состояние
2. Найти элемент img в галерее
3. Извлечь URL из атрибута src или data-src
4. Сохранить URL
5. Если не последнее изображение:
   - browser_click("Следующий слайд")
   - browser_wait_for(time: 1) - ждать загрузки
```

### Шаг 4: Анализ каждого изображения

Для каждого URL изображения:

```
1. Использовать Gemini Vision API для анализа
2. Применить промпт для описания изображения
3. Получить детальное описание
4. Сохранить результат: {index, url, description}
```

**Промпт для Gemini Vision:**

```
Проанализируй это изображение и опиши его детально:

1. Что изображено на фото? (интерьер, экстерьер, план, деталь)
2. Какие помещения/зоны видны?
3. Какой стиль оформления?
4. Какие материалы и отделка использованы?
5. Какие детали интерьера/мебели присутствуют?
6. Какое настроение создает изображение?
7. Какие особенности или уникальные элементы?

Опиши максимально подробно, как будто описываешь для потенциального покупателя.
```

### Шаг 5: Форматирование результатов

Вернуть результаты в структурированном виде:

```json
{
  "tab": "STANDARD",
  "total_images": 5,
  "images": [
    {
      "index": 1,
      "url": "https://example.com/image1.jpg",
      "description": "Интерьер гостиной в стиле минимализм..."
    },
    {
      "index": 2,
      "url": "https://example.com/image2.jpg",
      "description": "Кухня с островом..."
    }
  ]
}
```

## Полный пример промпта для Gemini

```
Задача: Проанализировать все изображения в галерее на вкладке STANDARD.

URL: https://innovatory-club.ru/katalog#!/tab/1063728081-3

Шаги выполнения:

1. Навигация:
   browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")

2. Ожидание загрузки:
   browser_wait_for(time: 3)

3. Получение снимка:
   browser_snapshot()

4. Определение количества изображений:
   - Найти индикатор "X из Y" в snapshot
   - Извлечь число Y (общее количество)

5. Для каждого изображения (от 1 до Y):
   a. Получить текущее изображение: browser_snapshot()
   b. Извлечь URL изображения из snapshot
   c. Проанализировать через Gemini Vision с промптом описания
   d. Сохранить: {index, url, description}
   e. Если не последнее - кликнуть "Следующий слайд" и подождать 1 сек

6. Вернуть все результаты в формате JSON

Важно:
- Анализировать ТОЛЬКО изображения из галереи вкладки
- Игнорировать логотипы, иконки, декоративные элементы
- Использовать детальные промпты для Gemini Vision
- Ждать загрузки после каждого переключения слайда
```

## Чек-лист для проверки

Перед возвратом результата убедись:

- [ ] Определено количество изображений из индикатора
- [ ] Извлечены URL всех изображений галереи
- [ ] Каждое изображение проанализировано через Gemini Vision
- [ ] Описания детальные и информативные
- [ ] Результаты структурированы (index, url, description)
- [ ] Игнорированы не относящиеся к галерее изображения

## Типичные проблемы и решения

### Проблема: Не могу найти индикатор "X из Y"

**Решение:**
- Искать альтернативные форматы: "X of Y", "X/Y"
- Прокрутить галерею и найти максимальное число
- Кликать "Следующий слайд" до неактивной кнопки

### Проблема: Изображения не загружаются

**Решение:**
- Увеличить время ожидания после клика (до 2 секунд)
- Проверить lazy loading (использовать data-src)
- Прокрутить страницу для загрузки изображений

### Проблема: Анализируются не те изображения

**Решение:**
- Фильтровать по контейнеру галереи
- Исключать элементы по размеру (маленькие = иконки)
- Исключать элементы по позиции (вне галереи)
