# Универсальный анализ URL для Gemini

## Описание

Этот документ описывает универсальный подход к анализу любых веб-страниц через Gemini с использованием браузерных инструментов и выделением смысловых блоков.

## Принцип работы

```
URL → Gemini (browser_navigate) → browser_snapshot → Извлечение смысловых блоков → Структурированный JSON
```

## Ключевые особенности

1. **Универсальность**: Подход работает для любого URL, любого сайта
2. **Автоматическая обработка**: Gemini сама обрабатывает JavaScript и динамический контент
3. **Фильтрация**: Автоматически игнорируется служебная информация (навигация, хедер, футер)
4. **Структурирование**: Результат возвращается в структурированном JSON формате

## Использование

### Базовый промпт для Gemini

```
Проанализируй страницу [URL] и выдели смысловые блоки, игнорируя служебную информацию.

1. Открой страницу через browser_navigate([URL])
2. Подожди загрузки JavaScript (browser_wait_for)
3. Получи snapshot страницы (browser_snapshot)
4. Извлеки смысловые блоки согласно инструкциям
5. Верни результат в JSON формате
```

### Что извлекается

- ✅ Заголовки (H1-H6)
- ✅ Описательные тексты
- ✅ Списки (маркированные, нумерованные)
- ✅ Таблицы с данными
- ✅ Цены и стоимость
- ✅ Галерея изображений (> 100x100px)
- ✅ Характеристики и преимущества
- ✅ Контакты

### Что игнорируется

- ❌ Навигация (role: navigation)
- ❌ Хедер (role: banner)
- ❌ Футер (role: contentinfo)
- ❌ Формы (если не нужны)
- ❌ Кнопки навигации
- ❌ Иконки и декоративные элементы (< 100x100px)

## Примеры использования

### Пример 1: Анализ страницы продукта

```
Пользователь: "Проанализируй https://example.com/product"

Gemini:
1. browser_navigate("https://example.com/product")
2. browser_wait_for(time: 3)
3. browser_snapshot()
4. Извлечь смысловые блоки
5. Вернуть JSON
```

### Пример 2: Анализ страницы с вкладками

```
Пользователь: "Проанализируй https://example.com/catalog#tab-1"

Gemini:
1. browser_navigate("https://example.com/catalog#tab-1")
2. browser_wait_for(time: 3)  # Ждать активации вкладки
3. browser_snapshot()
4. Извлечь только контент активной вкладки
5. Вернуть JSON
```

## Формат результата

```json
{
  "page_url": "https://example.com/page",
  "page_title": "Название страницы",
  "semantic_blocks": [
    {
      "type": "heading",
      "level": 1,
      "text": "Основной заголовок"
    },
    {
      "type": "text",
      "content": "Описательный текст..."
    },
    {
      "type": "list",
      "title": "Характеристики",
      "items": ["...", "..."]
    }
  ],
  "ignored_blocks": ["Навигация", "Хедер", "Футер"]
}
```

## Преимущества подхода

1. **Работает с любым сайтом**: Не зависит от структуры конкретного сайта
2. **Обрабатывает JavaScript**: Динамический контент загружается и анализируется
3. **Автоматическая фильтрация**: Служебная информация игнорируется автоматически
4. **Структурированный результат**: Данные возвращаются в удобном формате
5. **Адаптивность**: Подстраивается под структуру любой страницы

## Важные замечания

- Всегда используй `browser_wait_for` после `browser_navigate` для загрузки JavaScript
- Фильтруй изображения по размеру (> 100x100px) для исключения иконок
- Извлекай только из `role: main, article, section`
- Игнорируй элементы с `role: navigation, banner, contentinfo`
- Адаптируйся под структуру конкретной страницы, но следуй основным правилам
