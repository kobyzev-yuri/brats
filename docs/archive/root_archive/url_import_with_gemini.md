# Импорт по URL через Gemini (без парсера)

## Текущая проблема

**Текущий подход (неправильный):**
```
URL → Парсер → Извлечение текста → Библиотекарь → Gemini
```

**Проблемы:**
- Парсер не учитывает динамический контент (JavaScript)
- Парсер не видит активную вкладку из URL
- Парсер смешивает контент всех вкладок
- Теряется структура и контекст

## Правильный подход

**Новый подход (правильный):**
```
URL → Gemini (browser_navigate) → browser_snapshot → Извлечение блоков → KB
```

**Преимущества:**
- ✅ Gemini сама обрабатывает URL через браузер
- ✅ Выполняется JavaScript, видна активная вкладка
- ✅ Можно извлекать только нужный контент
- ✅ Сохраняется структура и контекст

## Алгоритм импорта по URL

### Шаг 1: Gemini получает URL

```
Пользователь: "Импортируй страницу https://innovatory-club.ru/katalog#!/tab/1063728081-3"
```

### Шаг 2: Gemini обрабатывает URL напрямую

```python
# Gemini использует browser инструменты напрямую
browser_navigate("https://innovatory-club.ru/katalog#!/tab/1063728081-3")
browser_wait_for(time: 3)  # Ждать выполнения JavaScript
browser_snapshot()  # Получить актуальное состояние страницы
```

### Шаг 3: Gemini извлекает контент

```python
# Gemini сама извлекает смысловые блоки из snapshot
blocks = extract_semantic_blocks(snapshot, active_tab="STANDARD")
```

### Шаг 4: Gemini обрабатывает через "библиотекаря"

```python
# Gemini обрабатывает извлеченные блоки
processed_blocks = process_through_librarian(blocks)
```

### Шаг 5: Gemini записывает в KB

```python
# Gemini записывает в KB с проверкой на дубликаты
save_to_kb(processed_blocks)
```

## Сравнение подходов

### ❌ Старый подход (с парсером):

```
URL → Парсер (извлекает весь текст) → Библиотекарь → Gemini → KB

Проблемы:
- Парсер не видит активную вкладку
- Парсер смешивает контент всех вкладок
- Парсер не выполняет JavaScript
- Теряется структура
```

### ✅ Новый подход (Gemini напрямую):

```
URL → Gemini (browser_navigate) → browser_snapshot → Извлечение блоков → Библиотекарь → KB

Преимущества:
- Gemini видит активную вкладку
- Gemini извлекает только нужный контент
- JavaScript выполняется
- Сохраняется структура
```

## Интеграция с "библиотекарем"

"Библиотекарь" - это обработка извлеченных блоков:

```python
def process_through_librarian(blocks):
    """Обработка блоков через библиотекаря"""
    processed = []
    
    for block in blocks:
        # Нормализация
        normalized = normalize_block(block)
        
        # Структурирование
        structured = structure_block(normalized)
        
        # Добавление метаданных
        structured['metadata'] = {
            'source_url': url,
            'tab': active_tab,
            'extracted_at': datetime.now(),
            'processed_by': 'librarian'
        }
        
        processed.append(structured)
    
    return processed
```

## Резюме

**Ответ на вопрос:**

Да, Gemini должна сама обрабатывать URL без парсера, используя browser инструменты напрямую:

- ✅ `browser_navigate(URL)` - навигация на страницу
- ✅ `browser_wait_for(time: 3)` - ожидание JavaScript
- ✅ `browser_snapshot()` - получение актуального состояния
- ✅ Извлечение блоков из snapshot
- ✅ Обработка через библиотекаря
- ✅ Запись в KB

**Преимущества:**
- Видит активную вкладку из URL
- Выполняет JavaScript
- Извлекает только нужный контент
- Сохраняет структуру

**Не нужен:**
- ❌ Внешний парсер для извлечения текста
- ❌ Промежуточная обработка HTML
- ❌ Статический парсинг без JavaScript
