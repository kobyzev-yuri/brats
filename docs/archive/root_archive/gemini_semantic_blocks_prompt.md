# Промпт для Gemini: Выделение смысловых блоков (универсальный)

## Полный промпт

```
КОНТЕКСТ: Парсинг любой веб-страницы для извлечения смысловой информации.

ЗАДАЧА: Извлечь ТОЛЬКО смысловые блоки, игнорируя служебную информацию.

ПРАВИЛА ИЗВЛЕЧЕНИЯ:

1. СМЫСЛОВЫЕ БЛОКИ (извлекать):
   - Основной заголовок страницы (H1, H2)
   - Заголовки разделов (H2, H3, H4)
   - Описательные тексты (параграфы, абзацы)
   - Списки (маркированные, нумерованные)
   - Таблицы с данными
   - Цены и стоимость (если есть)
   - Галерея изображений (только контентные изображения > 100x100px)
   - Контакты (адрес, телефон, email, график работы)
   - Характеристики, преимущества, особенности (если есть)
   - Любая другая релевантная информация для пользователя

2. СЛУЖЕБНЫЕ БЛОКИ (игнорировать):
   - Навигационное меню (role: navigation)
   - Хедер с логотипом (role: banner)
   - Футер (role: contentinfo)
   - Формы резервирования (если не нужны)
   - Кнопки навигации ("Предыдущий слайд", "Следующий слайд", "Закрыть")
   - Иконки и декоративные элементы (< 100x100px)
   - Служебные тексты: "Skip to main content", "Закрыть диалоговое окно", "Навигационное меню"

3. ПРАВИЛА ФИЛЬТРАЦИИ:
   - Извлекать только из role: main, article, section
   - Игнорировать элементы с role: navigation, banner, contentinfo
   - Игнорировать изображения < 100x100px (иконки, логотипы)
   - Игнорировать элементы вне основного контента
   - Игнорировать формы и модальные окна (если не нужны)

4. ИДЕНТИФИКАЦИЯ БЛОКОВ:
   - Заголовки: искать элементы H1-H6 в основном контенте
   - Описание: искать параграфы и текстовые блоки после заголовков
   - Списки: искать ul, ol элементы с релевантным содержимым
   - Таблицы: искать table элементы с данными
   - Цены: искать паттерны цен (цифры с валютой, "руб", "$", "€" и т.д.)
   - Контакты: искать заголовки "Контакты", "Contact", паттерны телефонов, email (@), адреса
   - Галерея: искать контейнеры с изображениями, исключать маленькие изображения
   - Характеристики: искать заголовки "Характеристики", "Преимущества", "Features", "Specifications" и списки после них

5. ФОРМАТ РЕЗУЛЬТАТА:
   Вернуть структурированный JSON:
   {
     "page_url": "...",
     "page_title": "...",
     "semantic_blocks": [
       {
         "type": "heading",
         "level": 1,
         "text": "Основной заголовок"
       },
       {
         "type": "text",
         "content": "Описательный текст..."
       },
       {
         "type": "list",
         "title": "Заголовок списка (если есть)",
         "items": ["Элемент 1", "Элемент 2"]
       },
       {
         "type": "table",
         "title": "Заголовок таблицы (если есть)",
         "data": [...]
       },
       {
         "type": "price",
         "value": "...",
         "currency": "..."
       },
       {
         "type": "gallery",
         "images": [
           {"url": "...", "alt": "...", "description": "..."}
         ]
       },
       {
         "type": "contacts",
         "address": "...",
         "phone": "...",
         "email": "...",
         "schedule": "..."
       },
       {
         "type": "features",
         "title": "Преимущества/Характеристики",
         "items": [...]
       }
     ],
     "ignored_blocks": ["Навигация", "Хедер", "Футер", "Формы", "Кнопки"]
   }

ВАЖНО:
- Извлекай ТОЛЬКО информацию, которая имеет смысл для пользователя
- Игнорируй всю служебную информацию
- Структурируй результат по типам блоков
- Указывай, какие блоки были проигнорированы
```

## Краткая версия промпта

```
Извлеки смысловые блоки из страницы, игнорируя служебную информацию:

ИЗВЛЕКАТЬ: заголовки (H1-H6), описания, списки, таблицы, цены, галерея, контакты, характеристики
ИГНОРИРОВАТЬ: навигация (role: navigation), хедер (role: banner), футер (role: contentinfo), формы, кнопки, иконки

Правила:
- Только из role: main, article, section
- Изображения > 100x100px
- Структурировать в JSON по типам блоков

Формат:
{
  "page_url": "...",
  "page_title": "...",
  "semantic_blocks": [...],
  "ignored_blocks": [...]
}
```

## Пример использования

### Запрос пользователя:
```
Проанализируй страницу https://example.com/product и выдели смысловые блоки, игнорируя служебную информацию.
```

### Действия Gemini:

```
1. browser_navigate("https://example.com/product")
2. browser_wait_for(time: 3)  # Ждать выполнения JavaScript
3. browser_snapshot()  # Получить актуальное состояние страницы

4. Извлечь смысловые блоки из snapshot:
   - Основной заголовок (H1)
   - Описательные тексты (параграфы)
   - Списки характеристик/преимуществ
   - Цены (если есть)
   - Галерея изображений (> 100x100px)
   - Контакты (если есть)
   - Таблицы с данными (если есть)

5. Игнорировать:
   - Навигационное меню (role: navigation)
   - Хедер с логотипом (role: banner)
   - Футер (role: contentinfo)
   - Формы (если не нужны)
   - Кнопки навигации
   - Иконки и декоративные элементы (< 100x100px)

6. Структурировать результат в JSON по типам блоков

7. Вернуть структурированный JSON результат
```

## Чек-лист для проверки

Перед возвратом результата:

- [ ] Извлечены только смысловые блоки из основного контента
- [ ] Игнорированы навигация (role: navigation), хедер (role: banner), футер (role: contentinfo)
- [ ] Игнорированы формы и кнопки навигации (если не нужны)
- [ ] Игнорированы иконки и декоративные элементы (< 100x100px)
- [ ] Извлечены все заголовки (H1-H6) из основного контента
- [ ] Извлечены описательные тексты и параграфы
- [ ] Извлечены списки с релевантной информацией
- [ ] Извлечены таблицы (если есть)
- [ ] Извлечены цены (если есть)
- [ ] Извлечена галерея (только контентные изображения > 100x100px)
- [ ] Извлечены контакты (если есть)
- [ ] Результат структурирован в JSON по типам блоков
- [ ] Указаны проигнорированные блоки

## Резюме

**Главное правило:**
Извлекай ТОЛЬКО информацию, которая имеет смысл для пользователя,
игнорируй всю служебную информацию (навигация, меню, формы, декоративные элементы).

**Ключевые моменты:**
- ✅ Извлекай из role: main, article, section
- ✅ Игнорируй role: navigation, banner, contentinfo
- ✅ Фильтруй изображения по размеру (> 100x100px)
- ✅ Структурируй результат по типам блоков (heading, text, list, table, price, gallery, contacts, features)
- ✅ Адаптируйся под структуру любой страницы
- ✅ Извлекай только релевантную информацию для пользователя
