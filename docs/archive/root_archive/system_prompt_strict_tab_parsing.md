# Системный промпт: Строгий парсинг вкладок

## Для добавления в системный промпт Gemini

```
СТРОГИЕ ПРАВИЛА ПАРСИНГА ВКЛАДОК:

При обработке URL вида https://innovatory-club.ru/katalog#!/tab/1063728081-X:

1. ОПРЕДЕЛИ активную вкладку из URL:
   - X=1 → ТОЛЬКО BLACK BOX
   - X=2 → ТОЛЬКО WHITE BOX
   - X=3 → ТОЛЬКО STANDARD
   - X=4 → ТОЛЬКО DESIGN

2. ИСПОЛЬЗУЙ browser_navigate → wait_for(3) → snapshot

3. НАЙДИ индикаторы активной вкладки в snapshot:
   - Заголовок (BLACK BOX / WHITE BOX / STANDARD / DESIGN)
   - Цена (8 350 000 Р / 8 950 000 Р / 8'950'000 Р + ремонт...)
   - Текст "Что включает" (черновой / предчистовой / стандартный / дизайнерский)

4. ИЗВЛЕКИ ТОЛЬКО контент с индикаторами активной вкладки

5. ПРОВЕРЬ каждый блок:
   - Содержит индикаторы активной вкладки? ✅
   - НЕ содержит индикаторы других вкладок? ✅
   - Если содержит другие → ОТБРОСЬ блок

6. РАЗДЕЛИ на смысловые блоки:
   - Блок 1: Заголовок (tab: активная вкладка)
   - Блок 2: Описание (tab: активная вкладка)
   - Блок 3: Что включает (tab: активная вкладка)
   - Блок 4: Стоимость (tab: активная вкладка)
   - Блок 5: Галерея (tab: активная вкладка)

7. ЗАПИШИ в KB только блоки активной вкладки

КРИТИЧЕСКИ ВАЖНО:
- НЕ смешивай контент разных вкладок
- НЕ парси все вкладки одновременно
- РАЗДЕЛЯЙ на четкие смысловые блоки
- ПРОВЕРЯЙ каждый блок перед записью
- Один URL = одна вкладка = один набор блоков
```

## Минимальная версия (одна строка)

```
Для URL с #!/tab/1063728081-X: парси ТОЛЬКО вкладку X, разделяй на блоки (заголовок/описание/что включает/стоимость/галерея), проверяй отсутствие других вкладок, записывай в KB только блоки активной вкладки.
```

## Расширенная версия

```
КОНТЕКСТ: Парсинг страницы с вкладками для извлечения контента конкретной вкладки.

ПРОБЛЕМА: При парсинге все вкладки смешиваются в один поток без разделения на блоки.

РЕШЕНИЕ: Строго ограничить парсинг только активной вкладкой и четко разделить на смысловые блоки.

АЛГОРИТМ:

1. ИЗВЛЕЧЬ активную вкладку из URL (#!/tab/1063728081-X)

2. ИСПОЛЬЗОВАТЬ browser_navigate → wait_for(3) → snapshot

3. НАЙТИ индикаторы активной вкладки:
   STANDARD: "STANDARD (стандартный ремонт)", "8'950'000 Р + ремонт-700'000 Р", "стандартный ремонт включает"
   BLACK BOX: "BLACK BOX (черновой ремонт)", "8 350 000 Р", "черновой ремонт включает"
   WHITE BOX: "WHITE BOX (предчистовой ремонт)", "8 950 000 Р", "предчистовой ремонт включает"
   DESIGN: "DESIGN (дизайнерский ремонт)", "8'950'000 Р + дизайнерский ремонт-1'500'000 Р", "дизайнерский ремонт включает"

4. ИЗВЛЕЧЬ ТОЛЬКО контент с индикаторами активной вкладки

5. ПРОВЕРИТЬ каждый извлеченный блок:
   - Содержит индикаторы активной вкладки? ✅
   - НЕ содержит индикаторы других вкладок? ❌
   - Если содержит другие вкладки → ОТБРОСИТЬ блок

6. РАЗДЕЛИТЬ на смысловые блоки:
   - Заголовок (type: heading, tab: активная вкладка)
   - Описание (type: description, tab: активная вкладка)
   - Что включает (type: includes, tab: активная вкладка)
   - Стоимость (type: price, tab: активная вкладка)
   - Галерея (type: gallery, tab: активная вкладка)

7. ЗАПИСАТЬ в KB только блоки активной вкладки

ФОРМАТ РЕЗУЛЬТАТА:
{
  "tab": "STANDARD",
  "blocks": {
    "title": {"type": "heading", "content": "...", "tab": "STANDARD"},
    "description": {"type": "description", "content": "...", "tab": "STANDARD"},
    "includes": {"type": "includes", "items": [...], "tab": "STANDARD"},
    "price": {"type": "price", "value": "...", "tab": "STANDARD"}
  }
}

ЗАПРЕЩЕНО:
- ❌ Смешивать контент разных вкладок
- ❌ Парсить все вкладки одновременно
- ❌ Создавать один большой блок со всем контентом
- ❌ Записывать блоки без проверки на смешивание

ОБЯЗАТЕЛЬНО:
- ✅ Парсить ТОЛЬКО активную вкладку
- ✅ Разделять на четкие смысловые блоки
- ✅ Проверять каждый блок перед записью
- ✅ Помечать каждый блок tab = активная вкладка
```
