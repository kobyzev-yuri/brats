# Краткие вопросы к заказчику

Документ для быстрого старта проекта. Можно отвечать в свободной форме или прямо в этом файле.

---

## 1. База знаний для нейропродажника

- **Контент**
  - Какие материалы можем использовать для обучения бота:
    - скрипты продаж,
    - примеры переписок с клиентами,
    - описания объектов и посёлка,
    - FAQ и типовые возражения?
  - В каком виде они есть сейчас (файлы, CRM, мессенджеры)?
  - За какой период есть история диалогов?

- **Правила общения**
  - Какие темы боту **нельзя** затрагивать?
  - Есть ли внутренние стандарты общения с клиентами?
  - В каких случаях бот обязан передать диалог менеджеру?

---

## 2. Данные и интеграции

- **Сайт и Метрика**
  - Адрес сайта (если есть другие проекты — перечислите).
  - ID счётчика Яндекс.Метрики и тип доступа, который вы можете выдать.
  - Какие формы/действия на сайте для вас самые важные (заявки, калькулятор, звонки и т.п.)?

- **amoCRM**
  - Поддомен amoCRM и контакт для создания интеграции.
  - Какие стадии воронки и основные источники лидов вы используете?
  - Есть ли уже интеграции (сайт, мессенджеры, телефония), которые важно не сломать?

- **Каналы общения**
  - Через какие каналы сейчас общаетесь с лидами:
    - Telegram, WhatsApp, Avito, телефон, email, другие?
  - Какие из этих каналов **обязательно** подключать к системе на первом этапе?

---

## 3. Бизнес‑процесс и критерии успеха

- **Как сейчас работает воронка**
  - Кратко: путь клиента от первого контакта до сделки (этапы).
  - На каких этапах чаще всего теряются лиды?
  - В какой момент лид считается “горячим” и передаётся менеджеру?

- **Цели и метрики**
  - Какие 2–3 показателя для вас самые важные:
    - конверсия в заявку/встречу/сделку,
    - скорость реакции на лида,
    - стоимость лида и т.п.?
  - Есть ли текущие значения этих метрик (хотя бы “на уровне ощущения”)?

---

## 4. Ограничения, безопасность, инфраструктура

- **Персональные данные**
  - Есть ли внутренние требования по 152‑ФЗ (размещение серверов, хранение логов и ПД)?
  - Можно ли использовать российские облачные LLM (ГигаЧат, YandexGPT), или нужна только полностью локальная модель?

- **Инфраструктура**
  - Где предпочтительно разворачивать систему:
    - ваши сервера,
    - наше облако,
    - ваш облачный провайдер?
  - Кто со стороны заказчика будет ответственным за доступы и согласование настроек?

---

## 5. Сроки, приоритеты и формат работы

- **Сроки**
  - Есть ли жёсткие даты (запуск рекламы, сезон, мероприятия), к которым важно успеть?

- **Приоритеты функционала**
  - Что важнее в первую очередь:
    - отчётность и аналитика,
    - прототип чат‑бота на одном канале,
    - интеграция с amoCRM и сайтом?
  - Какой минимальный функционал вы считаете MVP?

- **Коммуникация**
  - Контактные лица по проекту (кто решает по данным, по бизнесу, по ИТ).
  - Удобный формат взаимодействия (созвоны, мессенджер, email) и желаемая частота статусов.

---

**Просим сначала ответить на блоки 1–3 — они критичны для запуска разработки.**  
Остальные детали можем уточнять параллельно в процессе пилота.
