# Статус конфигурации amoCRM

## ✅ Что уже настроено

1. **AMOCRM_SUBDOMAIN** = `innovatoryclub`
   - Поддомен вашего аккаунта amoCRM

2. **AMOCRM_ACCESS_TOKEN** = (долгосрочный токен)
   - Срок действия: до 2026-04-30
   - Готов к использованию для API запросов

3. **AMOCRM_REFRESH_TOKEN** = (токен обновления)
   - Для автоматического обновления Access Token при необходимости

4. **AMOCRM_CLIENT_ID** = `297a49fd-3236-411a-941c-c8ff40bc7f44`
   - ID интеграции в amoCRM

## ⚠️ Что может понадобиться (опционально)

### 1. AMOCRM_CLIENT_SECRET

**Нужен если:**
- Хотите настроить OAuth2 credentials в n8n для автоматического обновления токенов
- Планируете использовать встроенные ноды AmoCRM в n8n (если они требуют OAuth2)
- Нужна автоматическая ротация токенов через OAuth2 API

**Где найти:**
- В карточке интеграции в amoCRM (amoМаркет → ваша интеграция)
- Показывается вместе с Client ID при создании интеграции

**Если не нужен:**
- Можно работать напрямую с API используя долгосрочный Access Token
- HTTP Request ноды в n8n могут использовать `{{$env.AMOCRM_ACCESS_TOKEN}}`
- Python-код может использовать `os.getenv("AMOCRM_ACCESS_TOKEN")`

### 2. AMOCRM_REDIRECT_URI

**Нужен если:**
- Настраиваете OAuth2 flow через n8n
- Используете встроенные OAuth2 credentials в n8n

**Обычно:**
- Для локальной разработки: `http://localhost:5678/rest/oauth2-credential/callback`
- Для продакшена: `https://your-n8n-domain.com/rest/oauth2-credential/callback`

**Если не нужен:**
- При использовании долгосрочного токена напрямую (как сейчас)

## 📋 Рекомендации

### Вариант 1: Использование долгосрочного токена (текущий)

**Достаточно того, что есть:**
- ✅ AMOCRM_SUBDOMAIN
- ✅ AMOCRM_ACCESS_TOKEN
- ✅ AMOCRM_REFRESH_TOKEN (на случай, если токен истечет)
- ✅ AMOCRM_CLIENT_ID (для справки)

**Как использовать:**
- В n8n workflow через HTTP Request:
  ```
  URL: https://{{$env.AMOCRM_SUBDOMAIN}}.amocrm.ru/api/v4/leads
  Headers:
    Authorization: Bearer {{$env.AMOCRM_ACCESS_TOKEN}}
  ```
- В Python-коде:
  ```python
  import os
  subdomain = os.getenv("AMOCRM_SUBDOMAIN")
  token = os.getenv("AMOCRM_ACCESS_TOKEN")
  ```

**Преимущества:**
- Простота настройки
- Не требует OAuth2 flow
- Токен действителен до 2026 года

### Вариант 2: OAuth2 через n8n (если нужна автоматическая ротация)

**Понадобится дополнительно:**
- ⚠️ AMOCRM_CLIENT_SECRET
- ⚠️ AMOCRM_REDIRECT_URI

**Как использовать:**
- Настроить OAuth2 credentials в n8n
- Использовать встроенные ноды AmoCRM (если доступны)
- Автоматическое обновление токенов через n8n

**Преимущества:**
- Автоматическое обновление токенов
- Интеграция с встроенными нодами n8n
- Более безопасное управление токенами

## 🎯 Вывод

**Для текущей работы достаточно того, что есть!**

Долгосрочный токен позволяет работать с API amoCRM без дополнительных настроек. Client Secret понадобится только если:
1. Вы хотите настроить OAuth2 credentials в n8n
2. Планируете использовать встроенные ноды AmoCRM в n8n
3. Нужна автоматическая ротация токенов через OAuth2 API

**Рекомендация:** Начните с текущей конфигурации (долгосрочный токен). Если понадобится автоматическое обновление токенов или интеграция с встроенными нодами n8n — добавьте Client Secret.
