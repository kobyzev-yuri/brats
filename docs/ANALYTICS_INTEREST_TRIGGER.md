# Интерес к объекту: как аналитик отслеживает и триггерит нейропродажника

## Два способа передачи «интереса» нейропродажнику

| Способ | Кто вызывает агента | Когда |
|--------|----------------------|--------|
| **Прямой (тестовая страница сейчас)** | Фронт | Сразу при клике «Интересуюсь этим объектом»: событие в аналитику + открытие чата и первое сообщение в n8n с контекстом объекта. |
| **Через аналитика** | Сервис sales-analytic | После того как накоплены события, кто-то вызывает `POST /api/analytics/trigger`; аналитик считает `intent_score` и при score ≥ 0.7 сам дергает sales-agent `/api/chat/initiate` и передаёт контекст (в т.ч. объект из `place_interest`). |

---

## Как аналитик отслеживает интерес

События приходят в **наш счётчик** (без Яндекс.Метрики): фронт шлёт их в `POST /api/analytics/collect` (или через прокси `http://localhost:8000/api/analytics-proxy`). Они пишутся в таблицу **analytics_events** с `source = 'site_integration'`.

### Учитываемые типы событий (расчёт intent_score)

- **place_interest** — клик «Интересуюсь этим объектом» на карточке. Сильный сигнал: **+0.75** к score, одной такой клик даёт «горячего» лида (порог 0.7).
- **card_view_duration** — долгий просмотр карточки объекта (если фронт присылает такое событие после N секунд на карточке). **+0.4** к score.
- **phone_click**, **whatsapp_click** — **+0.6** каждый.
- **form_submit** — **+0.5**, **phone_input** — **+0.4**, **email_input** — **+0.3**.
- **Время на сайте** > 5 минут — **+0.1** (по разнице времени первого и последнего события сессии).
- **Глубина** > 5 разных page_url — **+0.1**.

Итоговый score обрезается по 1.0. Если **score ≥ 0.7** (порог по умолчанию), лид считается «горячим» и при вызове триггера аналитик инициирует нейропродажника.

### «Время на карточке» / задержка мыши на картинке

На тестовой странице **catalog-interest-test.html** добавлен блок «Варианты ремонта» с картинками (черновая, стандартная, дизайнерский, предчистовая). Как только пользователь **задерживает мышь на картинке 10 секунд**, фронт отправляет событие `card_view_duration` с полями `place_id`, `place_name`, `object_name`, `duration_seconds`, `renovation_type`. Далее при включённой опции «Сначала аналитик» вызывается триггер; аналитик передаёт в sales-agent контекст **card_view_duration** (вариант ремонта). То же срабатывает при клике по кнопке «Интересуюсь этим объектом» на карточке объекта (событие `place_interest`).

## Как аналитик триггерит нейропродажника

1. Кто-то вызывает **POST /api/analytics/trigger** с параметрами:
   - `visitor_id`, `session_id` (обязательно),
   - `threshold=0.7` (опционально).
2. Аналитик по событиям этой сессии за последний час считает **intent_score** и список **reasons**.
3. Если **score ≥ threshold**:
   - подставляются контакты и контекст из последнего события (phone, email, page_url, form_type);
   - из последнего события типа **place_interest** вытаскивается контекст объекта (place_id, place_name, settlement, price, area);
   - отправляется запрос в **SALES_AGENT_URL/api/chat/initiate** с телом:
     - `external_id`, `channel`, `contact`, `context` (в т.ч. `intent_score`, `reasons`, **place_interest**).
4. Ответ (conversation_id и т.д.) возвращается в ответе триггера. Дальше фронт или n8n могут открыть чат с этим conversation_id или отправить первое сообщение в n8n webhook с тем же external_id и контекстом.

Вызов триггера может быть:
- **вручную** (curl / Postman) для проверки;
- по **расписанию** (n8n/cron) раз в N минут по активным сессиям;
- **с фронта** через несколько секунд после события (например, после `place_interest`): страница вызывает `/api/analytics/trigger`, и если `triggered: true`, открывает чат или редирект.

## Как протестировать

### 1. Тест «прямого» сценария (без аналитика)

- Запустить сайт с прокси и n8n:  
  `./start_all_services.sh start` (или только site + n8n).
- Открыть: **http://localhost:8000/catalog-interest-test.html**
- Нажать **«Интересуюсь этим объектом»** на любой карточке.
- Ожидание: открывается чат, уходит первое сообщение в n8n с контекстом объекта (metadata.real_estate_context), в чате приходит ответ агента. Аналитик в этой цепочке не участвует.

### 2. Тест цепочки «аналитик → триггер нейропродажника»

Нужны: **sales-analytic** (порт 8002), **sales-agent** (8003), БД с таблицей **analytics_events**.

1. Запустить аналитик и sales-agent (и при необходимости сайт с прокси):
   - БД: `DATABASE_URL` в config.env для sales-analytic.
   - В config.env для sales-analytic задать `SALES_AGENT_URL=http://localhost:8003`.
2. Открыть **http://localhost:8000/catalog-interest-test.html**.
3. Включить галочку **«Сначала аналитик: вызвать триггер через 2 сек»** (цепочка: событие → аналитик → нейропродажник).
4. URL аналитики оставить пустым при открытии с :8000 (подставится прокси); триггер при :8000 идёт через **/api/analytics-trigger-proxy**.
5. Нажать **«Интересуюсь этим объектом»** на карточке:
   - событие **place_interest** уйдёт в аналитику;
   - через 2 сек страница сама вызовет триггер;
   - в чате появится сообщение «Аналитик зафиксировал интерес (score 0.75)» или «Триггер не сработал», затем отправится первое сообщение в n8n.
6. **Вручную** (если нужно проверить без галочки): подставить свой visitor_id и session_id (в консоли: `localStorage.getItem('brats_visitor_id')`, `sessionStorage.getItem('brats_session_id')`) и вызвать:
   ```bash
   curl -X POST "http://localhost:8002/api/analytics/trigger?visitor_id=VISITOR_ID&session_id=SESSION_ID"
   ```
   Ожидание в ответе: `"triggered": true`, `"intent_score": 0.75`, `"reasons": ["Интерес к объекту (карточка/место)"]`, при настроенном SALES_AGENT_URL — `"sales_agent_called": true`, в теле запроса к sales-agent в `context` будет **place_interest** с полями объекта.

### 3. Узнать visitor_id и session_id для curl

На тестовой странице в консоли браузера (F12):

```javascript
localStorage.getItem('brats_visitor_id')
sessionStorage.getItem('brats_session_id')
```

Их подставить в URL:  
`http://localhost:8002/api/analytics/trigger?visitor_id=...&session_id=...`

## Сводка

- **Отслеживание интереса:** по событиям в **analytics_events** (наш счётчик): в т.ч. **place_interest** (клик по кнопке на карточке) и при желании **card_view_duration** (время на карточке с фронта).
- **Триггер:** вызов **POST /api/analytics/trigger** (вручную, по расписанию или с фронта). Аналитик считает score и при score ≥ 0.7 вызывает sales-agent **/api/chat/initiate** и передаёт контекст, включая объект из последнего **place_interest**.
