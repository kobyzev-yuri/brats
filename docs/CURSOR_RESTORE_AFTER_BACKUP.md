# Восстановление конфигов Cursor после бэкапа с битыми файлами

Если при создании бэкапа часть файлов в `~/.cursor` или `~/.config/Cursor` уже была повреждена (битые блоки на диске), при переносе на новый диск эти повреждения сохраняются.

## Что проверено (на этой машине)

- **Целы:** `~/.cursor/argv.json`, `extensions/extensions.json`, `~/.config/Cursor/User/settings.json`, `keybindings.json`, `Preferences`, проектные данные в `~/.cursor/projects/`.
- **Повреждены:** много файлов в `~/.config/Cursor/User/History/` — это **Local History** (локальная история версий файлов). Ошибка вида `Unterminated string` в `entries.json` означает обрезанный файл при записи/копировании.
- **Пустые файлы:** часть нулевого размера в `History/` (например `vBJa.json`) и служебные LOCK/LOG — норма для кэшей; пустые файлы внутри папок истории — вероятно обрезанные снапшоты.

Local History не влияет на настройки Cursor, правила, навыки или открытые проекты. Потеряется только возможность откатиться к старым версиям файлов через «Local History» в проводнике/меню.

## Вариант 1: Очистить только Local History (рекомендуется)

Удалить папку с локальной историей. Cursor создаст её заново при следующих правках файлов.

```bash
# Закрой Cursor полностью, затем:
rm -rf ~/.config/Cursor/User/History
```

После следующего запуска Cursor будет работать без старых локальных версий файлов; новая история начнёт копиться заново.

## Вариант 2: Полный сброс данных Cursor (если есть глюки после варианта 1)

Если после очистки History что-то всё ещё ломается (краши, странное поведение):

```bash
# Закрой Cursor полностью.

# Сброс только данных из ~/.cursor (проекты, навыки, MCP — будут пересозданы)
mv ~/.cursor ~/.cursor.bak

# При необходимости — полный сброс и конфига приложения (настройки, тема, расширения)
# mv ~/.config/Cursor ~/.config/Cursor.bak
```

После запуска Cursor создаст новый `~/.cursor`. Настройки (если не трогали `~/.config/Cursor`) сохранятся. Из `~/.cursor.bak` потом можно вручную перенести нужное (например `skills-cursor/`, `projects/` по одному проекту).

## Вариант 3: Поиск битых JSON вручную

Если хочешь чинить точечно (например, только в других проектах):

```bash
# Найти все entries.json и проверить
find ~/.config/Cursor/User/History -name 'entries.json' -exec sh -c 'python3 -c "import json; json.load(open(\"$1\"))" 2>/dev/null || echo "BROKEN: $1"' _ {} \;
```

Битые можно удалять по одному или удалить всю родительскую папку (одна папка = история одного файла).

## После восстановления

1. Запустить Cursor, открыть проект.
2. Проверить: настройки (Settings), интерпретатор Python, тема.
3. Проверить чат/Composer (Ctrl+L / Ctrl+I).
4. При необходимости заново указать Python: **Settings → Python → Default Interpreter Path** (после переезда диск мог смениться, путь в `settings.json` уже поправлен на текущий anaconda).
