# База знаний (KB) для агента продаж

Кратко: назначение, структура, хранение и использование в потоке n8n → LLM.

---

## Назначение

KB обеспечивает нейропродажника:

- **Информацией о продукте** — дома, цены, характеристики, коммуникации, территория.
- **Скриптами продаж** и обработкой возражений.
- **Контактами и процедурами** — показы, график, запись на просмотр.
- **Единым тоном общения** (tone_of_voice) и правилами по целевой аудитории.

Поиск по запросу пользователя выполняется в kb-service; релевантные фрагменты (chunks) и при необходимости медиа подставляются в промпт LLM.

---

## Хранение

- **PostgreSQL** с расширением **pgvector**. Таблица `knowledge_base`: `content` (текст), `embedding` (вектор), `metadata` (JSONB).
- **Метаданные:** категория, подкатегория, целевая аудитория (end_buyer / realtor / both), теги, источник. Медиа — в `metadata.images` и `metadata.documents` (url, описание). Схема: `db-init/02_create_knowledge_base.sql`.
- **Поиск:** векторный поиск по embedding запроса (kb-service генерирует embedding через OpenAI-совместимый API) + при необходимости фильтрация по metadata.

---

## Категории контента

| Категория | Назначение |
|-----------|------------|
| product_info | Описание посёлка, домов, участков, коммуникаций. |
| sales_script | Сценарии диалога под этап воронки. |
| objection_handling | Ответы на возражения. |
| contacts | Контакты, график, процедуры записи. |
| pricing | Цены, условия оплаты, ипотека. |
| location | Расположение, инфраструктура. |
| tone_of_voice | Стиль и правила общения. |

В metadata задаются также `target_audience`, `priority`, `context.use_case` (greeting, qualification, proposal и т.д.) для более точного подбора чанков.

---

## Использование в потоке

1. **n8n** получает сообщение чата → передаёт в DLP (обезличивание) → отправляет обезличенный текст в **kb-service** `POST /api/kb/search`.
2. **kb-service** возвращает топ-N релевантных chunks и их metadata (в т.ч. images, documents).
3. Узел **Format KB Context** в n8n формирует блок «База знаний» и при наличии — блок «Релевантные медиа» для промпта LLM.
4. Ответ агента и приложенные медиа возвращаются в чат; чат отображает картинки и ссылки на документы по `response.media`.

До контакта KB используется для привлечения и ответов на общие вопросы; после контакта — те же чанки плюс память диалога, слоты календаря и логика КП/договора по [FUNNEL_CALENDAR_CP_CONTRACT.md](./FUNNEL_CALENDAR_CP_CONTRACT.md).

## Загрузка и админка

- **Скрипты:** `load_kb_from_data.py` (загрузка из `data/`), `reset_and_load_kb.py` (очистка и загрузка). Импорт по URL и сохранение через API библиотекаря (mmkb).
- **Админка KB** (Streamlit, порт 8501): поиск и редактирование chunks, редактирование блока медиа, импорт по URL.

Подробная структура метаданных и примеры чанков — в архиве: `docs/archive/` (KB_STRUCTURE, MULTIMODAL_KB и др.). Концепция и медиа в ответе: [CONCEPT.md](./CONCEPT.md).
