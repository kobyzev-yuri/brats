# Диагностика: нет записи в CRM, данные не доходят до n8n

## Если данные не уходят в n8n (чат не отвечает, в Executions пусто)

1. **Откройте страницу с одного адреса:**  
   **http://localhost:8000/site-integration-example.html**  
   (не с file:// и не с другого порта — иначе запрос может уходить не туда или блокироваться CORS.)

2. **Запустите прокси в терминале:**  
   ```bash
   cd /projects/brats
   python site-integration/serve_with_proxy.py
   ```  
   В настройках чата должен быть URL: **http://localhost:8000/api/n8n-proxy**. Тогда запросы идут на этот же хост (без CORS) и прокси пересылает их в n8n.

3. **Проверьте в том же терминале:** отправьте сообщение в чат — должна появиться строка вида  
   `[n8n] POST chat -> message='...' ... external_id='...'`  
   и затем `[n8n] POST chat <- 200`.  
   Если этих строк нет — запрос до прокси не доходит (проверьте, что открыли именно http://localhost:8000 и что в поле «URL n8n webhook» указан http://localhost:8000/api/n8n-proxy).

4. **Если используете прямой URL n8n** (например http://localhost:5678/webhook/sales-agent-kb): убедитесь, что n8n запущен, workflow «Sales Agent - KB Integration» включён (кнопка Active), в узле Webhook выбран метод POST. Иначе запрос вернёт 404 или соединение будет отклонено.

---

Если после отправки формы или сообщения в чате **в AmoCRM не появляется лид/контакт** и **в n8n «ничего не видно»** (запрос не приходит или цепочка не срабатывает), проверьте цепочку по шагам ниже.

---

## 1. Откуда идут запросы

| Действие | Куда уходит запрос | Что должно произойти |
|----------|--------------------|----------------------|
| **Отправка формы** (имя, телефон, email) | Сначала `POST .../api/test-lead-from-chat` (amocrm-api), затем открывается чат и туда отправляется первое сообщение | Лид и контакт создаются в AmoCRM, слоты (имя, телефон) пишутся в диалог |
| **Сообщение в чате** | `POST` на URL n8n webhook (или `/api/n8n-proxy`, если сайт с localhost:8000) | n8n получает тело `{ message, channel, external_id, metadata }`, запускает workflow, возвращает ответ |

Если форма или чат вызывают **другой URL** (не прокси и не amocrm-api), запросы могут уходить «в никуда» или на старый сервис — тогда в CRM и в n8n ничего не будет.

---

## 2. Проверка без видимых ошибок

### 2.1. Запуск через прокси (localhost:8000)

Запустите сервер с прокси из корня репо:

```bash
python site-integration/serve_with_proxy.py
```

В **этом же терминале** при отправке формы и при отправке сообщения в чате появятся строки:

- `[amocrm] POST test-lead-from-chat -> 200 created=True` — лид создан.
- `[amocrm] POST test-lead-from-chat -> 502` или `недоступен` — amocrm-api не ответил (проверьте порт 8010, config.env).
- `[n8n] POST chat -> message='...' ... external_id='...'` — запрос ушёл в n8n.
- `[n8n] POST chat <- 200` — n8n ответил успешно.
- `[n8n] POST chat <- HTTP 404` или `недоступен` — workflow не найден или n8n не запущен (порт 5678).

Если после нажатия «Отправить» в форме **ни одной строки** `[amocrm]` нет — запрос на создание лида с сайта **вообще не уходит**. Тогда смотрите п. 3.1.

Если после сообщения в чате **нет** `[n8n] POST chat` — запрос в n8n **не уходит** (неверный URL чата, другой backend). Смотрите п. 3.2.

### 2.2. Браузер: вкладка Network (F12)

1. Откройте DevTools (F12) → вкладка **Network**.
2. Отправьте форму или сообщение в чате.
3. Найдите запрос:
   - на **test-lead-from-chat** (форма) — смотрите **Status** и **Response**. 200 и `created: true` — лид создан; 4xx/5xx — скопируйте ответ и проверьте amocrm-api и config.env.
   - на **n8n-proxy** или на **webhook** (чат) — Status 200 и в ответе есть `agent_response` или `response` — n8n отработал.

Если запроса на test-lead-from-chat нет — форма дергает не тот URL (поле «URL FastAPI amoCRM» пустое и fallback не срабатывает, или страница открыта не с того хоста). Смотрите п. 3.1.

## 3. Частые причины

### 3.1. Запись в CRM не создаётся (форма)

- **Форма не вызывает amocrm-api.**  
  На странице примера при пустом поле «URL FastAPI amoCRM» используется fallback `origin + '/api/amocrm'`. Если вы открываете сайт **не** с того же хоста, где крутится прокси (например, файл открыт как `file://` или с другого порта без прокси), запрос уйдёт на этот origin и может дать 404.  
  **Решение:** открывать страницу с `http://localhost:8000/...` и держать запущенным `serve_with_proxy.py`, либо явно указать в настройках URL amocrm-api: `http://localhost:8010`.

- **amocrm-api не запущен или не доступен.**  
  **Решение:** `cd amocrm-api && uvicorn api.main:app --host 0.0.0.0 --port 8010`, проверить `curl -s http://localhost:8010/health`.

- **Ошибка AmoCRM (токен, права).**  
  В ответе на test-lead-from-chat будет 502 и текст ошибки. Проверьте config.env (AMOCRM_*, DATABASE_URL), логи amocrm-api.

### 3.2. Данные не доходят до n8n (чат)

- **Неверный URL n8n в настройках чата.**  
  Должен быть либо `http://localhost:8000/api/n8n-proxy` (если сайт с 8000), либо прямой URL webhook, например `http://localhost:5678/webhook/sales-agent-kb`. В консоли браузера уже есть лог: `[Чат] Отправка в n8n: <url> ...` — проверьте, что url правильный.

- **n8n не запущен или workflow выключен.**  
  **Решение:** открыть n8n (http://localhost:5678), включить workflow (кнопка Active), проверить, что у узла Webhook выбран метод POST и используется Production URL.

- **CORS.**  
  Если чат открыт с другого домена (не localhost:8000), браузер может блокировать запрос. Используйте прокси (открывать сайт с 8000) или настройте CORS в n8n.

### 3.3. n8n получил запрос, Executions есть, но в AmoCRM нет контакта (нет Василисы)

Лид создаётся цепочкой **Maybe Create Lead** → **Create Lead If Phone** → **AmoCRM Create Lead**. Пошагово:

1. **Откройте успешный Execution** (тот, где вы писали от имени Василисы с телефоном).
2. **Узел Maybe Create Lead** — откройте его вывод (Output):
   - Если **should_create: false** или **lead_payload: null** — телефон из сообщения не извлёкся. Тогда: переимпортируйте workflow из репозитория (`n8n/workflows/sales-agent-kb-integration-localhost.json`) — в нём обновлённая логика для «тел. 79106337788» и «Я оставил заявку: Имя Фамилия, тел. …». Либо вручную вставьте код из [n8n/MAYBE_CREATE_LEAD_EXTRACTION.md](../n8n/MAYBE_CREATE_LEAD_EXTRACTION.md).
   - Если **should_create: true** и в **lead_payload** есть **phone**, **name** — идите к шагу 3.
3. **Узел Create Lead If Phone** — ветка «да» должна была выполниться. Откройте следующий узел **AmoCRM Create Lead**:
   - Если узел **не выполнялся** (серый) — условие `should_create === true` не сработало (проверьте тип данных в If).
   - Если **AmoCRM Create Lead** выполнился с **ошибкой** — откройте его вывод: там будет ответ amocrm-api (502, 401, detail: «…»). Запустите amocrm-api на порту 8010, проверьте config.env (AmoCRM токены, DATABASE_URL). В n8n в переменных окружения задайте **AMOCRM_API_BASE_URL** (например `http://localhost:8010`); в узле AmoCRM Create Lead URL должен вести на этот адрес (если n8n в Docker — используйте `http://host.docker.internal:8010` или IP хоста).
   - Если узел выполнился **успешно** (зелёный), но в AmoCRM контакта нет — откройте вывод узла: там должен быть `lead_id`, `contact_id`, `created: true`. Если `created: false` — лид уже существовал для этого external_id. Если есть lead_id, но в AmoCRM пусто — проверьте тот же аккаунт AmoCRM и воронку (тестовые лиды с префиксом [BRATS-TEST]).

**Важно:** смотреть нужно вывод узла **«Maybe Create Lead»**, а не «Respond to Webhook» или «Prepare Response». В графе Execution кликните по узлу **Maybe Create Lead** и откройте вкладку **Output** — там должны быть поля **should_create** и **lead_payload**. Если в этом узле таких полей нет или они пустые — в n8n загружена старая версия workflow: **Workflows** → ⋯ → **Import from File** → выбрать `n8n/workflows/sales-agent-kb-integration-localhost.json` (заменить текущий), включить **Active**.

**Итог:** в Execution по шагам смотрите Maybe Create Lead (телефон извлёкся?) → AmoCRM Create Lead (запрос ушёл? ответ 200? lead_id в ответе?). Если телефон не извлекается — переимпортируйте workflow из файла в репозитории.

## 4. Краткий чеклист

1. Запущен ли **serve_with_proxy.py** (порт 8000) и смотрите ли вы логи в его терминале.
2. Открываете ли страницу с **http://localhost:8000/...** (чтобы форма и чат ходили через прокси).
3. Запущен ли **amocrm-api** (8010) и **n8n** (5678), workflow включён (Active).
4. В n8n в переменных задан **AMOCRM_API_BASE_URL** (например `http://localhost:8010`) для узлов Resolve и AmoCRM Create Lead.
5. В **Executions** n8n виден ли последний запуск и на каком узле ошибка (если есть).

После этого запись в CRM и прохождение данных в n8n можно отслеживать по логам прокси и по Network/Executions.
