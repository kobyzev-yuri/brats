# Личный кабинет (ЛК)

- **login.html** — вход по логину (email или телефон) и паролю (номер телефона). Учётные данные выдаются клиенту в чате после записи на просмотр (при создании контакта).
- **cabinet.html** — чат с агентом: КП, договор, технические детали ремонта и обустройства территории.

## Как открыть

При запуске **amocrm-api** (порт 8010) страницы раздаются с того же хоста:

- Вход: `http://localhost:8010/lk/login.html`
- После входа — перенаправление на `http://localhost:8010/lk/cabinet.html`

Убедитесь, что применена миграция **10_create_users_and_lk.sql** (таблицы `lk_users`, `lk_sessions`, колонка `conversations.user_id`).

## Выдача логина и пароля в чате

При вызове `POST /api/test-lead-from-chat` (создание контакта из чата) в ответе возвращаются поля:

- `lk_login` — логин для ЛК (email, если передан, иначе телефон)
- `lk_password_hint` — подсказка: «пароль: ваш номер телефона (как в заявке)»

Их можно подставить в ответ клиенту в чате, например:  
«Вы записаны. Для входа в личный кабинет: логин **…**, пароль — ваш номер телефона. Ссылка: https://…/lk/login.html»

## Отправка сообщений из ЛК

Сообщения из кабинета уходят в тот же диалог, что и с сайта. Для ответа агента нужен **N8N_WEBHOOK_URL** в config.env: amocrm-api вызывает этот webhook с `conversation_id` и `channel: "lk"`. Workflow n8n должен учитывать `conversation_id` и не создавать новый диалог.
