# Резюме: Созданные скрипты для обогащения KB

## Дата создания
2026-02-08

## Созданные скрипты

### ✅ 1. `kb_enrichment.py` - Основной скрипт обогащения KB

**Назначение:** Комплексное обогащение базы знаний из analytics_events

**Функциональность:**
- Обнаружение новых возражений
- Анализ популярных вопросов (через `analyze_questions.py`)
- Обновление информации о продуктах (через `update_product_info.py`)
- Улучшение скриптов продаж (через `improve_sales_scripts.py`)

**Использование:**
```bash
python scripts/kb_enrichment.py --days 30 --dry-run
```

---

### ✅ 2. `analyze_questions.py` - Анализ популярных вопросов

**Назначение:** Анализ вопросов клиентов и автоматическое добавление ответов в KB

**Функциональность:**
- Анализ событий `question_asked` за указанный период
- Группировка вопросов по частоте (минимум 5 раз)
- Проверка наличия ответов в KB
- Генерация ответов через LLM с использованием контекста из KB
- Автоматическое добавление в KB с правильными метаданными

**Использование:**
```bash
python scripts/analyze_questions.py --days 30 --min-frequency 5 --dry-run
```

---

### ✅ 3. `update_product_info.py` - Обновление информации о продуктах

**Назначение:** Обновление информации о продуктах в KB на основе аналитики просмотров

**Функциональность:**
- Анализ событий `page_view` для определения популярных объектов
- Анализ вопросов по конкретным объектам
- Проверка наличия информации в KB
- Автоматическое добавление информации о популярных объектах
- Обновление метаданных (добавление тега "популярный")

**Использование:**
```bash
python scripts/update_product_info.py --days 30 --min-views 10 --dry-run
```

---

### ✅ 4. `improve_sales_scripts.py` - Улучшение скриптов продаж

**Назначение:** Анализ успешных диалогов и автоматическое улучшение скриптов продаж

**Функциональность:**
- Анализ успешных диалогов (привели к КП)
- Выявление паттернов успешных сообщений по этапам
- Сравнение с текущими скриптами в KB
- Автоматическое улучшение скриптов на основе успешных паттернов
- Обновление скриптов в KB с версионированием

**Использование:**
```bash
python scripts/improve_sales_scripts.py --days 30 --dry-run
```

## Принципы реализации

### 1. Использование сервисов вместо хардкода
- Все скрипты используют `EmbeddingService` и `LLMService` из `kb-service`
- Модели читаются из `config.env`, а не хардкодятся
- См. [CONFIGURATION_REFERENCE.md](../../docs/CONFIGURATION_REFERENCE.md)

### 2. Поддержка dry-run режима
- Все скрипты поддерживают `--dry-run` для тестирования без изменений
- Позволяет проверить, что будет сделано, перед реальным выполнением

### 3. Обработка ошибок
- Graceful degradation при недоступности сервисов
- Подробное логирование ошибок
- Продолжение работы при ошибках в отдельных модулях

### 4. Модульность
- Каждый скрипт может работать независимо
- Основной скрипт `kb_enrichment.py` использует остальные как модули
- Легко расширять и добавлять новые функции

## Конфигурация

Все скрипты используют переменные из `config.env`:

**Обязательные:**
- `DATABASE_URL` - подключение к PostgreSQL

**Опциональные:**
- `KB_API_URL` - URL KB Service API (по умолчанию `http://localhost:8001`)
- `OPENAI_API_KEY`, `OPENAI_BASE_URL`, `OPENAI_MODEL` - для LLM
- `EMBEDDING_PROVIDER`, `EMBEDDING_MODEL` или `HF_MODEL_NAME` - для embeddings

**См. также:**
- [CONFIGURATION_REFERENCE.md](../../docs/CONFIGURATION_REFERENCE.md) - полный справочник по конфигурации
